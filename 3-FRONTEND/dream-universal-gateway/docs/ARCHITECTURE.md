# Dream Universal Gateway — 通用AI入口架构方案

> 版本: v2.5 | 日期: 2026-05-14 | 作者: Dream Architecture Team
> **v2.5核心变更**: 新增用户体系(UserSystem)，邮箱注册登录→分配UID→UID关联API配置/可用资金/杠杆/策略等，形成用户配置列表，支撑批量用户
> **v2.4核心变更**: 新增积分体系(CreditsManager)模块，策略执行前积分余额检查，不足暂停/充值恢复
> **v2.3核心变更**: 自定义策略输入从结构化选择器改为自然语言+意图识别，新增回测验证模块
> **v2.2核心变更**: 新增策略设置模块(StrategyConfig)，四大独立功能：API配置+交易参数+策略设置+通信渠道
> **v2.1核心变更**: 新增交易参数配置模块(TradingParams)，API配置与交易参数分离，通信渠道独立

## 1. 项目定位与核心价值

### 1.1 一句话定义

**通用AI对话入口** — 一个聪明的金融AI助手，核心能力是理解用户意图、智能检索已有产物、高效路由处理，而非简单消息收发。

### 1.2 三层架构总览

```
┌──────────────────────────────────────────────────────────┐
│  Layer 1: 通用AI前端入口 (NEW — 独立项目)                  │
│  ┌─────────────────────────────────────────────────────┐ │
│  │  对话界面+意图识别+产物路由+API配置+交易设置+策略设置+通信渠道+用户体系│ │
│  │  路径: /chat  |  技术栈: Next.js (独立项目，通过API调用产物中台)  │ │
│  │  认证: Auth.js v5 + PostgreSQL | UID关联全配置                    │ │
│  └────────────────────────┬────────────────────────────┘ │
│                           │ SSE + REST                    │
├───────────────────────────┼──────────────────────────────┤
│  Layer 2: 产物中台 (EXISTING — 内部监控+产物中转站)       │
│  ┌────────────────────────┴────────────────────────────┐ │
│  │  Next.js :3456 | /feed 产物浏览 | /api/* 数据接口    │ │
│  │  543+ 产物 | 14 部门 | 分类索引                      │ │

│  └─────────────────────────────────────────────────────┘ │
└──────────────────────────────────────────────────────────┘
```

### 1.3 核心价值主张（v2.0 重新定义）

| 维度 | 说明 | 对比普通ChatGPT |
|------|------|-----------------|
| **意图即路由** | 说什么=做什么，系统自动识别并路由到最优处理路径 | 用户需手动选工具/模型 |
| **产物优先** | 先检索已有产物，不重跑全链路，节省90%+资源 | 每次从零分析 |
| **增量智能** | 只补充变化部分，而非推倒重来 | 每次完整重算 |
| **黑箱透明** | 用户看到分析过程但不暴露内部代号 | 要么黑箱要么暴露技术细节 |
| **金融通用** | 当前聚焦BTC，但框架适用于任何金融品种 | 品种绑定 |

### 2.2 技术选型

| 层 | 技术 | 理由 |
|----|------|------|
| Layer 1 前端 | Next.js 14 + React 18 + Tailwind | **同技术栈独立项目**，不共享代码库，通过API调用产物中台 |
| Layer 1 路由 | `/chat` (独立项目) | 独立Next.js项目，通过REST/SSE与产物中台交互 |
| Layer 1 状态 | Zustand (轻量) | 比 Redux 简单，比 Context 性能好 |
| Layer 1 渲染 | Server Component + Client Component 混合 | 首屏SSR，交互区CSR |
| Layer 2 API | Next.js Route Handlers | SSE流式 + REST |
| **意图引擎** | **自研 IntentRouter** | **核心差异化，非通用ChatBot** |
| 链路执行 | 内部执行器(A0-A9映射) | 用户不可见，按需执行 |
| 配置存储 | 本地加密文件 (AES-256) | API Key 等敏感信息不上云，本地加密存储 |

### 2.3 关键设计决策（v2.0更新）

| 决策 | 选择 | 理由 |
|------|------|------|
| 系统核心是什么？ | **意图识别+产物路由** | 不是链路可视化，是智能路由 |
| 用户看到A系列吗？ | **否，完全隐藏** | 用户说"分析"，不说"A2" |
| 每次都跑完整链路？ | **否，产物优先+增量更新** | 节省90%资源，响应更快 |
| 斜杠命令用什么名？ | **用户直觉命名** | `/分析`而非`/A2` |
| 进度展示怎么写？ | **用户语言描述过程** | "正在深度分析"而非"A2执行中" |
| SSE还是WebSocket？ | **SSE** | 对话场景单向流即可，实现简单 |
| 配置存储在哪？ | **本地加密文件** | API Key敏感，不上传云端 |

## 5. 术语映射表

**系统内部所有使用A系列的地方，对外展示必须经过此映射表转换。**

| 内部术语 | 用户可见术语 | 使用场景 |

## 7.5 交易参数配置功能（摘要）

> 详见 [TRADING_CONFIG_DESIGN.md](TRADING_CONFIG_DESIGN.md) ⭐ **v2.1新增**

### 7.5.1 核心定位

交易API设置 ≠ API密钥管理。配置交易所API后，系统根据A系列策略自动操作，**但交易行为必须在用户设定的安全边界内执行**。

### 7.5.2 交易参数项

| 参数 | 说明 | 当前约束 |
|------|------|----------|
| 交易金额 | 按余额百分比交易，系统统一定 | 默认10%，所有用户一致 |
| 交易类型 | 现货/合约 | 默认仅现货 |
| 杠杆倍数 | 允许的杠杆范围 | 1x-5x |
| 最大日亏损 | 绝对金额+百分比双维度 | 默认500U/5% |
| 最大账户亏损 | 绝对金额+百分比双维度 | 默认2000U/20% |
| 交易品种 | 允许的品种 | 默认仅BTC |

### 7.5.3 下单门禁 (TradingGate)

策略触发交易后，系统**必须**检索用户交易参数列表，按UID查询配置，下单前进行门禁过滤：

```
A5决策输出 → TradingGate(查询UID配置→8步校验) → 通过/拦截
```

门禁为必须项，不可跳过。拦截时通过通信渠道推送告警。

## 7.6 策略设置功能（摘要）

> 详见 [STRATEGY_CONFIG_DESIGN.md](STRATEGY_CONFIG_DESIGN.md) ⭐ **v2.2新增**

### 7.6.1 核心定位

策略设置是第四大独立功能，解决"执行什么"的问题。推荐策略由A4定时推送，自定义策略聚焦**技术指标信号策略**（用户自然语言描述→意图识别→策略库匹配→A系列调优→回测验证）。

### 7.6.2 策略分类

| 策略类型 | 来源 | 触发方式 | 用户操作 |
|----------|------|----------|----------|
| **推荐策略** | A4验证定时任务自动生成 | 系统推送 | 查看→应用/忽略 |
| **信号策略** | 用户自然语言描述→意图识别→策略库匹配→A系列调优→回测 | 用户自然语言输入 | 查看→确认应用/存为草稿/下载 |

### 7.6.3 信号策略核心流程

自定义策略(信号策略)聚焦技术指标（RSI/MACD/MA/BOLL等），用户用自然语言描述策略意图，系统自动识别解析后执行完整流程：

```
用户自然语言("RSI低于30并且MACD金叉时做多") → 意图识别解析(IntentParser) → 策略库匹配模板(85%) → A系列分析调优 → 回测验证(Backtester) → 输出信号策略
```

技术指标和策略库**不易漂移**，定时执行效果更优。自动化执行前必须通过回测验证（胜率≥45%|回撤≤20%|PF≥1.2），不达标则A系列二次调优(最多1次)。

### 7.6.4 策略应用 → 定时任务

用户选择策略并确认应用后，系统根据用户UID下的API配置创建定时执行任务：

```
策略应用 → 前置检查(API+交易参数) → 策略参数适配 → 创建定时任务(1h/4h/1d) → 自动执行
```

每次定时执行时：获取最新A4产物 → 条件匹配 → TradingGate门禁 → 下单/跳过 → 通知

## 10. 安全设计

### 10.1 API Key 安全

| 措施 | 说明 |
|------|------|
| 加密存储 | AES-256-GCM，密钥派生自主密码+设备指纹 |
| 传输安全 | 前端→后端仅HTTPS，后端→交易所仅HTTPS |
| 内存安全 | 密钥仅在内存中解密，不落盘明文 |
| 前端脱敏 | 显示 `••••••kT9x`，全密钥不传输到前端 |
| 审计日志 | 每次API调用记录时间+操作，不含密钥值 |

### 10.2 通信渠道安全

| 措施 | 说明 |
|------|------|
| Token存储 | Bot Token 同等加密保护 |
| 消息过滤 | 不推送API Key等敏感字段 |
| 推送确认 | P0级推送需用户确认后才执行交易 |
| 频率限制 | 每渠道每分钟最多10条，防刷屏 |
