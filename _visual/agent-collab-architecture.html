3) 最小闭环状态推进

目前控制器只承诺自动推进三段：accepted→ledgered→archived→knowledge_synced。 其余状态由人/协议驱动。

stateDiagram-v2 [*] --> accepted accepted --> ledgered: VALIDATION_RESULT\nDecision=ACCEPTED\nHandoff=ledgered ledgered --> archived: VALIDATION_RESULT\nHandoff=archived archived --> knowledge_synced: VALIDATION_RESULT\nHandoff=knowledge_synced\nclosure完整 knowledge_synced --> [*]

4) 触发与硬阻断点（最容易出“逻辑漏洞”的地方）

  • Task ID 缺失：控制器直接 BLOCK（无法定位任务）。
  • Governance Handoff 缺失：控制器 BLOCK（无法得知 requested_status）。
  • 账本缺 Task：BLOCK（RULE_TASK_NOT_FOUND）。
  • shared-sync 且 Current Sync State=pending 但无 DESIGN_REVIEW：BLOCK。
  • 幂等性：重复触发应 PASS 且不重复追加 reward（已做）。
  • 权限：workflow 写回 main 需要 contents: write（已改）。

5) PR9 任务树如何支撑多 AGENT 协作（文档真源挂载）

任务的 sync_checkpoint 直接挂设计真源文件（例如 COMPANY_CENTRAL_HUB + 7-ARTIFACT-HUB-V2 子设计文档）， 使协作不依赖“口头记忆”。
