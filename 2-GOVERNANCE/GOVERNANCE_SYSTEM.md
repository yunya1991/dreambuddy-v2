## 🏛️ 系统概述

中台治理系统是Dream-MultiSkill的合规保障体系，负责：
- **合规审查**: AI交易合规、策略变更审查
- **门禁管理**: 交易前门禁、提案影子验证
- **违规处理**: 违规类型定义、处罚执行
- **审计追踪**: 决策链路固化、审计证据
- **秘书服务**: 邮件路由、产物投递、日报管理

## 🔧 核心SKILL

### 1. 治理管理部 (dream-governance-manager)

**核心功能**:
- 四步法流程执行
- 违规类型判定
- 处罚执行
- 治理文档管理

**目录结构**:
```
~/.workbuddy/skills/dream-governance-manager/
├── SKILL.md                      # SKILL定义
├── governance_docs/              # 治理文档
│   ├── CHARTER.md               # 治理章程
│   ├── MANUAL.md               # 操作手册
│   ├── DEPARTMENT_RULES.md     # 部门规则
│   ├── SKILL_STANDARDS.md      # SKILL标准
│   └── VIOLATION_TYPES.md      # 违规类型
├── compliance_audit/            # 合规审计
│   ├── audit_checklist.md
│   └── audit_report_*.md
├── penalties/                   # 处罚记录
│   └── penalty_records.md
└── automations/                # 自动化配置
```

**四步法流程**:
```
Step1: FAQ自愈
   ↓ 检查 FAQ/ 目录标准答案
Step2: 治理文档检索
   ↓ 检索 governance_docs/
Step3: 联网搜索
   ↓ 使用 tavily 外部搜索
Step4: 自主分析
   ↓ 基于已有知识推理
```

### 2. AI交易合规审查 (ai-trading-compliance)

**核心功能**:
- 策略变更合规审查
- PASS/WARN/FAIL判定
- 灰度发布管理
- 审计日志

**审查维度**:
| 维度 | 描述 | 权重 |
|------|------|------|
| 风险合规 | 风险敞口、杠杆限制 | 30% |
| 策略合规 | 策略逻辑、参数边界 | 25% |
| 执行合规 | 下单频率、仓位限制 | 25% |
| 文档合规 | 记录完整性、追溯性 | 20% |

**输出格式**:
```json
{
  "result": "PASS|WARN|FAIL",
  "risk_score": 0-100,
  "issues": [
    {
      "code": "RISK_001",
      "severity": "HIGH|MEDIUM|LOW",
      "description": "问题描述",
      "recommendation": "修复建议"
    }
  ],
  "audit_trail": {
    "timestamp": "ISO8601",
    "reviewer": "ai-trading-compliance",
    "version": "v2.0"
  }
}
```

### 3. 交易前门禁 (dream-pretrade-gatekeeper)

**核心功能**:
- 数据完整性检查
- 评分冲突检测
- 账户回撤熔断
- 执行成本评估

**门禁检查项**:
```
✅ 数据完整性
   ├── 价格数据时效 <5min
   ├── 持仓数据同步
   └── 评分数据有效

✅ 评分冲突
   ├── 多模型评分差异 <20%
   └── 无反向信号

✅ 账户熔断
   ├── 24h回撤 <5%
   ├── 账户余额充足
   └── 无异常标记

✅ 执行成本
   ├── 预估手续费 <0.1%
   ├── 预估滑点 <0.05%
   └── 市价单 vs 限价单判断
```

### 4. 影子验证门 (hermes-shadow-verification-gate)

**核心功能**:
- Proposal影子验证
- 指标劣化检测
- 验证结果输出

**验证流程**:
```
1. 接收 Proposal
        ↓
2. 生成影子执行
        ↓
3. 对比验证指标
        ↓
4. 输出 PASS/REJECT
        ↓
5. 生成劣化解释
```

### 5. SKILL治理门 (hermes-skill-governance)

**核心功能**:
- SKILL命名规范
- 范围冲突检测
- 发布约束校验

### 6. 回滚执行器 (hermes-rollback-actuator)

**核心功能**:
- 变更定位
- 回滚补丁生成
- 回滚后校验
- 审计证据输出

### 7. 盘后MRM审计 (dream-posttrade-mrm-audit)

**核心功能**:
- 决策输入快照
- 执行偏差分析
- 结果归因
- Episode固化

## 🔄 合规审查流程

### 策略变更审查

```
用户提交变更
        ↓
ai-trading-compliance 审查
        ↓
    ┌───┴───┐
    ↓       ↓
  PASS    FAIL/WARN
    ↓       ↓
执行   hermes-shadow-verification-gate
    ↓       ↓
    └───┬───┘
        ↓
    验证结果
        ↓
    ┌───┴───┐
    ↓       ↓
  PASS    REJECT
    ↓       ↓
执行   返回修改建议
```

### 交易执行门禁

```
A5执行请求
        ↓
dream-pretrade-gatekeeper
        ↓
    ┌───┴───┐
    ↓       ↓
  PASS    FAIL
    ↓       ↓
执行   返回SKIP
        ↓
  触发A7门禁
```
