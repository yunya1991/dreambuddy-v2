## 3. 改进建议

### 3.1 架构改进建议

#### 3.1.1 组件拆分与复用
**当前问题**: `dashboard/page.tsx` 295行，所有组件都在一个文件中。

**改进建议**:
```
src/components/
├── layout/
│   ├── Sidebar.tsx           # 左侧栏
│   ├── ChatArea.tsx          # 中间对话区
│   ├── RightPanel.tsx        # 右侧面板
│   └── UserProfileCard.tsx   # 用户配置概览卡片
├── chat/
│   ├── MessageList.tsx       # 消息列表
│   ├── MessageBubble.tsx    # 消息气泡
│   ├── InputArea.tsx         # 输入区域
│   └── QuickCommands.tsx    # 快捷命令
├── settings/
│   ├── ApiConfigPanel.tsx    # API配置
│   ├── TradingConfigPanel.tsx # 交易设置
│   ├── StrategyPanel.tsx     # 策略设置
│   └── ChannelPanel.tsx     # 通信渠道
└── ui/
    ├── DataCard.tsx          # 数据卡片
    ├── AnalysisProgress.tsx  # 分析进度
    └── ArtifactRefCard.tsx  # 参考报告卡片
```

#### 3.1.2 状态管理优化
**当前问题**: 使用 useState 管理状态，随着功能增加会变得难以维护。

**改进建议**:
- 使用 Zustand 或 Redux Toolkit 进行全局状态管理
- 按功能模块拆分 store:
  - `auth-store.ts` - 用户认证状态
  - `chat-store.ts` - 对话和消息状态
  - `config-store.ts` - API/交易/策略配置
  - `ui-store.ts` - UI状态（面板折叠、主题等）
  - `credits-store.ts` - 积分状态

#### 3.1.3 API集成层
**当前问题**: Dashboard中直接调用API，缺少统一的API集成层。

**改进建议**:
```typescript
src/lib/
├── api/
│   ├── client.ts             # API客户端（统一错误处理）
│   ├── auth.ts              # 认证相关API
│   ├── user.ts              # 用户相关API
│   ├── chat.ts              # 对话相关API
│   ├── config.ts            # 配置相关API
│   └── credits.ts          # 积分相关API
└── types/
    ├── api.ts              # API响应类型
    ├── user.ts             # 用户类型
    └── chat.ts             # 对话类型
```

### 3.2 UI/UX改进建议

#### 3.2.1 加载状态优化
**当前问题**: 加载中只显示"加载中..."，用户体验不佳。

**改进建议**:
- 使用骨架屏（Skeleton）代替简单的loading文字
- 消息发送后显示"思考中..."动画（参考UI_SPEC.md §6.4）
- 分析进度使用进度条 + 阶段提示

#### 3.2.2 错误信息优化
**当前问题**: 错误信息直接显示技术术语，用户难以理解。

**改进建议**:
- 所有错误信息使用用户语言（参考UI_SPEC.md §9术语禁用清单）
- 提供"重试"按钮和"查看详情"选项
- 对于API错误，显示友好的提示和解决方案

#### 3.2.3 响应式设计
**当前问题**: 当前只实现了桌面端布局，缺少移动端适配。

**改进建议**:
- 按UI_SPEC.md §2.2实现响应式断点
- 移动端使用底部Tab栏代替左侧栏
- 触摸优化（按钮大小、间距）

### 3.3 性能优化建议

#### 3.3.1 消息列表虚拟化
**问题**: 当对话历史很长时，渲染所有消息会导致性能问题。

**建议**: 使用 `react-window` 或 `tanstack-virtual` 实现虚拟滚动。

#### 3.3.2 图片懒加载
**问题**: 参考报告中的图片会一次性加载。

**建议**: 使用 Intersection Observer API 实现图片懒加载。

#### 3.3.3 API请求优化
**问题**: 每次组件挂载都重新获取数据。

**建议**:
- 使用 SWR 或 React Query 进行数据缓存
- 实现乐观更新（Optimistic Update）
- 请求去重（Deduplication）
