## 🏛️ 系统定位

网关中台是Dream-MultiSkill的统一接入层，负责：
- **用户接入**: 邮箱登录/API配置/策略管理
- **交易接入**: OKX API配置/交易参数/策略设置
- **通信接入**: Telegram/邮件/Webhook多通道
- **积分管理**: 积分消耗/预算控制/配额管理

## 🔧 四大功能模块

### 1. API配置模块

**核心功能**:
- OKX API Key管理（加密存储）
- 多交易所支持（预留扩展）
- API状态监控
- 密钥轮换

**数据结构**:
```typescript
interface APIConfig {
  uid: string;
  exchange: 'okx' | 'binance' | 'huobi';
  apiKey: string;        // 加密存储
  secretKey: string;      // 加密存储
  passphrase: string;    // OKX专用
  mode: 'demo' | 'live';
  status: 'active' | 'inactive' | 'expired';
  createdAt: string;
  updatedAt: string;
}
```

### 2. 交易参数模块

**核心功能**:
- 账户选择（dreamdemo / A5）
- 杠杆设置（1x - 5x）
- 风险参数（止损/止盈/滑点）
- 仓位限制

**配置示例**:
```yaml
trading:
  profile: dreamdemo    # 账户
  leverage: 2           # 杠杆
  max_position: 0.1    # 最大仓位
  stop_loss: 2.0        # 止损 %
  take_profit: 5.0      # 止盈 %
  slippage: 0.1        # 滑点容忍 %
```

### 3. 策略设置模块

**核心功能**:
- 策略模板选择
- 参数自定义
- 回测验证
- 自动运行调度

**支持Regime**:
| Regime | 描述 | 适用策略 |
|--------|------|----------|
| TRENDING_UP | 上涨趋势 | 趋势跟随 |
| TRENDING_DOWN | 下跌趋势 | 做空策略 |
| RANGE_BOUND | 区间震荡 | 网格交易 |
| VOLATILE | 高波动 | 突破策略 |
| BREAKOUT | 突破 | 动量策略 |

### 4. 积分系统

## 🔐 安全设计

### 认证机制

```typescript
// JWT Token认证
interface AuthToken {
  uid: string;
  exp: number;
  iat: number;
}

// 授权流程
1. 用户登录 → 邮箱验证 → 生成JWT
2. JWT存储 → httpOnly Cookie / LocalStorage
3. 请求携带 → Authorization: Bearer {token}
```

### 数据加密

- API密钥: AES-256-GCM加密
- 密码: bcrypt哈希
- 传输: HTTPS/WSS

## 🔄 集成接口

### WorkBuddy桥接

**中台即时触发模式**:
```
用户请求 → Gateway API → task-manager.ts
                            ↓
                    POST /api/task
                            ↓
                    task-poller.py (异步)
                            ↓
                    执行结果 → 产物中台
```

**双模式对比**:
| 模式 | 适用场景 | 响应时间 |
|------|----------|----------|
| 🔗 中台即时 | 对话/查询任务 | 秒级 |
| 💬 直接执行 | 简单指令 | 即时 |
