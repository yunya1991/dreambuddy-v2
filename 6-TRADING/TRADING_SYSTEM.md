### 3.2.1 加仓阶梯预设原则 ★★ v2.1 核心原则

**核心原则**: 阶梯加仓 = 以更优价格买入

```
原则1: 最多3次加仓（4层仓位：初始 + 3次加仓）
原则2: 必须分阶梯预设（每层阶梯有明确价格间隔）
原则3: 阶梯基于波动率等指标计算（常规情景）
原则4: 机遇来临时动态调整（但不取消阶梯）
```

**阶梯预设方法**:

```
┌─────────────────────────────────────────────────────────────┐
│                  加仓阶梯预设（做多情景）                      │
├─────────────────────────────────────────────────────────────┤
│                                                              │
│  当前价 $80,000                                              │
│      │                                                      │
│      │  ─ ─ ─ 入场价 $79,800 (Level 0)                       │
│      │       └─ 初始建仓                                    │
│      │                                                      │
│      │  ─ ─ ─ 加仓1 $79,000 (Level 1)                        │
│      │       └─ 第一阶梯：基于波动率计算间隔                  │
│      │                                                      │
│      │  ─ ─ ─ 加仓2 $78,200 (Level 2)                       │
│      │       └─ 第二阶梯：继续下探更优价格                    │
│      │                                                      │
│      │  ─ ─ ─ 加仓3 $77,500 (Level 3)                       │
│      │       └─ 第三阶梯：极限低价（不轻易触发）              │
│      │                                                      │
└─────────────────────────────────────────────────────────────┘
```

**阶梯间隔计算**:
```
阶梯间隔 = max(波动率 × k, 最小间隔)

示例:
- BTC 价格 $80,000
- 20日波动率 3%
- k = 0.5（保守）
- 阶梯间隔 = max(80000 × 0.03 × 0.5, $500) = max($1200, $500) = $1200
```

| 加仓层 | 价格计算 | 说明 |
|--------|----------|------|
| Level 0 | 预设入场价 | 初始建仓 |
| Level 1 | 入场价 - 间隔1 | 第一阶梯 |
| Level 2 | Level 1 - 间隔2 | 第二阶梯（可扩大间隔） |
| Level 3 | Level 2 - 间隔3 | 极限阶梯（慎用） |

**动态调整规则**:
```
场景1: 机遇出现（如大幅下跌至Level 2）
→ 调整: 把Level 2变成新的Level 1
→ 保持: 阶梯结构不变（仍然有下一级）
→ 结果: 以更低成本买入，阶梯仍然存在

场景2: 市场快速下跌至Level 3
→ 判断: 是否是基本面恶化？
→ 是: 停止加仓，执行A9离场
→ 否: 保持Level 3，等待反弹

场景3: 市场反转上涨
→ 撤销未触发的加仓单
→ 等待下一轮回踩
```

**禁止事项**:
```
✗ 取消阶梯直接all-in
✗ 无限制加仓（超过3次）
✗ 在上一单未成交前挂下一单
✗ 追涨杀跌（上涨时取消加仓单）
```

**阶梯预设模板**:
```yaml
加仓计划:
  最大加仓次数: 3
  总仓位上限: 四等份原则(≤25%账户)
  
  阶梯1:
    价格: $79,000
    间隔: $800
    仓位占比: 8%
    触发条件: 回踩Level 1价格
    
  阶梯2:
    价格: $78,200
    间隔: $800
    仓位占比: 8%
    触发条件: 回踩Level 2价格
    
  阶梯3:
    价格: $77,500
    间隔: $700
    仓位占比: 4%
    触发条件: 极限回调（慎用）
```

### 3.4 A9紧急离场决策 ★ v2.2 新增 / v2.2 核心修复

> **核心矛盾解决**: L1止损必须等三层加仓完成后才完整启用，否则三层永远无法加满

**触发条件**: 检测到重大风险事件

```

```

**A9四层离场** ★ v2.2 优化:

| 层级 | 类型 | 说明 | 核心约束 |
|------|------|------|----------|
| **L1** | 止盈止损 | TP/SL价格触发 | ★ 加仓期：止损仅硬性保护；持仓期：三层加满后启用 |
| **L2** | 信号反转 | 日线信号反转/Regime变化 | 立即离场，不等待 |
| **L3** | 风险事件 | 21事件库触发 | 强制离场，黑天鹅/政策突变 |
| **L4** | 参数止损 | 时间止损/最大回撤 | 最后防线 |

**执行优先级**: L1 > L2 > L3 > L4

**★ L1止损分层执行机制**:

```
┌─────────────────────────────────────────────────────────────┐
│  阶段1: 加仓期（三层未满）                                   │
│  ─────────────────────────────                              │
│  • 止损作为【硬性保护】，不轻易触发                          │
│  • 阈值放宽至跌破Level 3极限价位才触发                       │
│  • 目的: 让三层加仓有足够空间完成                            │
├─────────────────────────────────────────────────────────────┤
│  阶段2: 持仓期（三层加满后）                                 │
│  ─────────────────────────────                              │
│  • 止损正式启用                                              │
│  • 基于加仓后均价计算止损位                                  │
│  • 止盈动态调整：                                            │
│    - 常规行情：不调整，保持预设                              │
│    - 趋势加强：提高止盈目标20-50% ★优势                     │
│    - 强信号达标：立即止盈，不追求最高点                      │
└─────────────────────────────────────────────────────────────┘
```

**★ 止盈动态调整规则**（相对于传统马丁策略的优势）:

| 趋势状态 | 止盈调整 | 说明 |
|----------|----------|------|
| 常规行情 | 不调整 | 保持预设，不频繁变动 |
| 趋势加强 | 提高止盈 | 扩大目标20-50% |
| 趋势衰竭 | 提前止盈 | 适当降低，快速离场 |
| 强信号达标 | 立即止盈 | 不追求最高点 |

### 3.5 信号过滤机制 ★

**原则**: 不是所有信号都需要响应

**信号分类与处理**:

| 信号强度 | 示例 | 处理方式 |
|----------|------|----------|
| **强信号** | 突破关键位、基本面重大变化 | ✓ 执行调整 |
| **弱信号** | 超时、价格噪音、轻微偏离 | ✗ 忽略 |
| **需确认信号** | 中等强度信号 | 等更强信号或时间确认 |

**过滤规则**:
```
if 信号强度 < 阈值:
    if 无更强技术信号支撑:
        if 无基本面变化:
            → 忽略（等待）
    else:
        → 执行调整
```

**判断标准**:

| 维度 | 强信号标准 | 弱信号标准 |
|------|-----------|------------|
| 技术面 | 突破均线/形态确认 | 轻微偏离、噪音 |
| 基本面 | 宏观数据/政策变化 | 新闻噪音 |
| 量能 | 放量确认 | 缩量/无量 |
