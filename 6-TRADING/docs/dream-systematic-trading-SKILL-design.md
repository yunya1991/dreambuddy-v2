## 3. 第一屏：周线决策 (战略层)

### 3.1 核心功能

> **⚠️ 核心定位**: 第一屏是**战略层**，使用**周线级别A1-A3** + 技术分析（周线）+ 基本面 → 输出:**策略类型 + 方向**

**执行原则**:
- ✅ 只做方向判断和策略选择
- ❌ 不预设价格
- ❌ 不下单

### 3.2 分析流程 (周线级别 A1-A3)

```yaml
第一屏分析流程 (周线级别 A1-A3):

  ================================================================
  【周线级别 A1-A3 流水线】★ 核心分析工具
  ================================================================
  
  输入:
    # 周线技术分析
    - Elder 三屏周线数据 (EMA趋势、MACD、周线结构)
    - 周线支撑阻力位
    - 周线成交量分析
    
    # 基本面分析
    - 宏观经济环境 (利率、通胀、美元走势)
    - 行业/赛道基本面 (存储赛道: NAND/SSD供需)
    - ETF 资金流数据 (周级别)
    
    # 链上 + 市场情绪
    - 链上数据 (BTC/ETH/SOL 周线持仓变化)
    - 期权 IV 周级别数据
    - Fear & Greed Index (周平均)
    
  处理: ★ 周线级别 A1-A3 ★
  ────────────────────────────────────────────────────────────────
    
  【A1 矛盾调查 (周线)】
    • 识别周线级别多空矛盾:
      - 供需矛盾: 周线供应区 vs 周线需求区
      - 多空矛盾: 多头驱动因素 vs 空头驱动因素
      - 宏观矛盾: 宏观经济 vs 行业基本面
    • 输出: 周线矛盾清单 + 矛盾优先级排序
    
  【A2 第一性原理 (周线)】
    • 分析周线趋势动力来源:
      - 趋势是沿着阻力最小方向运行
      - 周线趋势具有延续性
      - 识别周线级别的主要矛盾转化点
    • 输出: 周线阻力最小方向 + 趋势动力评级
    
  【A3 沙盘推演 (周线)】
    • 多情景周线推演:
      - 情景A: 多头延续 (概率？%)
      - 情景B: 空头延续 (概率？%)
      - 情景C: 震荡整理 (概率？%)
    • 输出: 周线多情景概率分布 + 最优策略路径
  ────────────────────────────────────────────────────────────────
  
  多空不对称性判断:
    情景A: 多头信号强 (周线评分 ≥ 70)
      → 选择多头策略
      → 可选: 多头合约马丁 / 多头现货马丁
      
    情景B: 空头信号强 (周线评分 ≥ 70)
      → 选择空头策略
      → 可选: 空头合约马丁 / 空头现货马丁
      
    情景C: 多空信号都弱 (周线评分 < 70)
      → 观望等待
      → 等待周线信号明确
      
  ================================================================
  【双轨策略选择】★ 第一屏核心输出
  ================================================================
  
  【合约马丁策略】
    适用场景:
      • 高波动市场 (周线波动率 > 5%)
      • 趋势明确但波动大 (周线 EMA 排列清晰)
      • 合约杠杆交易
      
    特点:
      • 使用合约（多空双向）
      • 带杠杆（可调整）
      • 收益放大，但风险同步放大
      
  【现货马丁策略】
    适用场景:
      • 低波动/震荡市场 (周线波动率 < 3%)
      • 长期持有为主
      • 稳健增值需求
      
    特点:
      • 使用现货（不支持做空）
      • 无杠杆或低杠杆
      • 风险可控，适合稳健策略
      
  策略选择矩阵:
    | 周线状态 | 推荐策略 | 理由 |
    |----------|----------|------|
    | 高波动 + 趋势明确 | 合约马丁 | 收益放大 |
    | 低波动 + 震荡市 | 现货马丁 | 稳健增值 |
    | 高波动 + 方向不明 | 观望 | 风险控制 |
    | 趋势初期 | 现货马丁 | 积累筹码 |
    | 趋势加速期 | 合约马丁 | 趋势跟随 |
      
  ================================================================
  【第一屏输出 → 第二屏输入】★ 关键传递
  ================================================================
  
  输出:
    - 周线方向: 做多 / 做空 / 观望
    - 策略类型: 合约马丁 / 现货马丁 ★
    - 周线信号强度评分 (0-100)
    - 周线情景类型: 情景A/B/C
    - 仓位上限建议
    - 风险预算分配
    - 有效周期: 本周 ~ 下周 (约1-2周)
    
  → 传递到第二屏:
    - 周线方向信号 ★
    - 策略类型 ★
    - 信号强度参考
```

## 4. 第二屏：日线预设 (战术层)

### 4.1 核心功能

> **⚠️ 核心定位**: 第二屏是**战术层**，使用**日线级别A1-A3**进行综合调研
> 
> **关键输入**: 第一屏输出的周线方向信号 + 策略类型（基于第一屏方向，不独立判断方向）
> 
> **三大预设**:
> 1. **入场时机预设** - 预设入场价格区间
> 2. **加仓价位预设** - 预设加仓价位表（最多3次）★ 阶梯预设原则
> 3. **止盈止损预设** - 预设止盈止损价位
> 
> **执行原则**: 制定详细计划，**预设完成即可直接在交易所挂限价单**，无需等待第三屏

### 4.2 分析流程 (日线级别 A1-A3)

```yaml
第二屏日线信号流程 (日线级别 A1-A3):

  ================================================================
  【日线级别 A1-A3 流水线】★ 核心分析工具
  ================================================================
  
  输入:
    # ★ 来自第一屏 (周线方向信号)
    - 周线方向: 做多 / 做空 / 观望
    - 策略类型: 合约马丁 / 现货马丁
    - 周线信号强度评分
    - 周线情景类型
    
    # 日线技术分析
    - 日线 Elder 三屏数据
    - 日线 EMA 排列
    - 日线 MACD / RSI
    - 20日波动率数据
    - 日线支撑阻力位
    
    # 其他数据
    - 昨日策略执行结果
    - 日内消息面信号
    
  处理: ★ 日线级别 A1-A3 ★
  ────────────────────────────────────────────────────────────────
    
  【A1 矛盾调查 (日线)】★ 基于第一屏方向
    • 在周线方向的框架下，识别日线级别矛盾:
      - 日线供需矛盾: 日内支撑位 vs 日内阻力位
      - 与周线方向的一致性检验
      - 日内驱动因素分析
    • 输出: 日线矛盾清单 + 与周线方向的共振/背离判断
    
  【A2 第一性原理 (日线)】★ 基于第一屏方向
    • 分析日线趋势动力:
      - 是否与周线方向共振
      - 日内阻力最小方向
      - 日内关键突破点识别
    • 输出: 日线阻力方向 + 关键价位 + 与周线一致性评级
    
  【A3 沙盘推演 (日线)】★ 基于第一屏方向
    • 多情景日线推演 (在周线方向约束下):
      - 情景1: 顺利执行 (价格按周线方向运行)
      - 情景2: 震荡回调 (日内波动，但不影响周线方向)
      - 情景3: 信号弱化 (日线与周线背离，需警惕)
    • 输出: 日线情景概率分布 + 关键价位预设
  ────────────────────────────────────────────────────────────────
  
  信号强度评估:
    - 强信号阈值: 日线评分 ≥ 70 + 与周线共振
    - 弱信号阈值: 日线评分 < 70 或 与周线背离
    
  ================================================================
  【核心输出】三大预设（预设完成 → 直接挂单）★
  ================================================================
  
  预设1: 入场时机预设 ★ v3.0 核心修改
    核心原则: 弱信号不下漏趋势 + 默认多头 + 强度值资金控制
    
    信号强度评估:
      • 强信号: 日线评分 ≥ 70 + 与周线共振 → 全仓入场
      • 中信号: 日线评分 50-69 或 与周线部分共振 → 半仓入场
      • 弱信号: 日线评分 < 50 或 与周线背离 → 轻仓试探（默认多头）
      • 无明确信号: 默认多头 + 最小仓位试探
    
    资金管理规则 ★ 核心创新:
      ──────────────────────────────────────────────────────────────
      仓位计算公式:
        实际仓位 = 基础仓位 × (信号强度 / 100) × 风险系数
        
      其中:
        • 基础仓位: 根据策略类型确定（合约马丁 20% / 现货马丁 25%）
        • 信号强度: 0-100 评分（日线评分 + 周线共振加成）
        • 风险系数: 0.5-1.5（根据波动率调整）
      ──────────────────────────────────────────────────────────────
    
    入场决策矩阵:
      | 信号强度 | 默认方向 | 仓位比例 | 入场方式 | 说明 |
      |----------|----------|----------|----------|------|
      | 强信号 (≥70) | 跟随周线 | 100% | 市价单立即入场 | 不等待回落 |
      | 中信号 (50-69) | 跟随周线 | 60% | 限价单分批入场 | 可等待小额回落 |
      | 弱信号 (<50) | 默认多头 | 30% | 限价单试探 | 不下漏趋势 |
      | 无信号 | 默认多头 | 10% | 限价单最小试探 | 保持市场存在感 |
    
    情景1 (弱信号/无信号):
      • ❌ 不再"等待回落" → 可能错过趋势
      • ✅ 改为"轻仓试探" → 默认多头 + 最小仓位
      • 示例: 弱信号 → 30%仓位 @ 当前价 ± 1%
      • 示例: 无信号 → 10%仓位 @ 当前价 ± 0.5%
    
    情景2 (强信号/中信号):
      • 强信号 → 全仓入场（100%仓位）
      • 中信号 → 半仓入场（60%仓位）
      • 可适当提前入场，不等待回落
      • 预设入场价格: 当前价 ± 1%
    
    ★ 关键改进:
      - 弱信号不再"等待回落"，而是"轻仓试探"
      - 无明确信号时"默认多头"（除非周线明确看空）
      - 仓位大小由"信号强度值"动态调整
    
  预设2: 加仓阶梯预设 ★★ 核心原则
    ──────────────────────────────────────────────────────────────
    原则1: 最多3次加仓（4层仓位：初始 + 3次加仓）
    原则2: 必须分阶梯预设（每层阶梯有明确价格间隔）
    原则3: 阶梯基于波动率等指标计算（常规情景）
    原则4: 监控出重要机遇则动态调整（但必须保持阶梯结构）
    ──────────────────────────────────────────────────────────────
    
    加仓阶梯间隔计算:
      阶梯间隔 = max(波动率 × k, 最小间隔)
      示例: BTC $80,000 × 3%波动率 × 0.5k = $1200间隔
    
    加仓阶梯示意（做多情景）:
      Level 0: $79,800 (入场)
      Level 1: $79,000 (间隔 $800)
      Level 2: $78,200 (间隔 $800)
      Level 3: $77,500 (间隔 $700, 极限阶梯)
    
    动态调整规则 ★ v2.8 核心进化:
    ──────────────────────────────────────────────────────────────
    核心原则: 基于价值评估做调整，追求更优价格
    
    ┌─────────────────────────────────────────────────────────┐
    │  评估类型      │  核心问题        │  决策依据              │
    ├─────────────────────────────────────────────────────────┤
    │  A4 价值验证  │  是否值得入场?    │  矛盾论验证 + 风险验证  │
    │  A5 价值入场  │  现在入场吗?      │  信号强度 + 时机判断    │
    │  A6 价值监控  │  有更优价格吗?    │  实时行情 + 阶梯结构保持│
    │  A9 风险离场  │  是否要离场?      │  21事件库 + 风险评估    │
    └─────────────────────────────────────────────────────────┘
    
    A4 价值评估 → 调整决策 ★★★★★:
      • A4 PASS → 确认阶梯有效，执行调整
      • A4 FAIL → 延迟调整，等待更好价格
      • A4 信号强度 → 影响调整幅度
      
    A5 价值评估 → 入场决策 ★★★☆☆:
      • A5 强信号 → 立即执行阶梯调整
      • A5 弱信号 → 等待A4确认后再调整
      
    A6 价值评估 → 时机监控 ★★★☆☆:
      • A6 发现更优价格 → 触发阶梯动态调整
      • A6 发现风险信号 → 触发A9风险评估
      
    A9 风险评估 → 离场决策 ★★★★★:
      • A9 强制离场: 基本面恶化 / 21事件触发
      • A9 条件离场: 移动止盈 / 时间止损
      • A9 风险预警 → 暂停阶梯调整，等待风险解除
    
  预设3: 止盈止损预设
    情景1 (弱信号):
      • 止损: 基于20日波动率和支撑阻力位
      • 止盈: 基于阻力位和波动率
      • 最多加仓3次，资金分四等份
      
    情景2 (强信号):
      • 调整入场时机、加仓位、止盈位和止损位
      • 信号强度影响调整幅度
      • 遵循系统宪法约束
    
    ★ v2.8 重要更新: A9 L1止损分层执行
      • 加仓期 (Level 0-2): 仅硬性保护止损（防止爆仓）
      • 三层加满后 (Level 3): 启用完整止盈止损
      • 止盈动态调整: 盈利超10% → 移动止盈到成本价
```

## 6. 实时监控机制详细设计

### 6.1 A6 情报监控系统

```yaml
A6 情报监控系统:

  监控频率:
    • 正常状态: 每小时运行
    • 异常状态: 每15分钟运行
    • 紧急状态: 实时监控
    
  监控指标:
    • 价格偏离度: (当前价 - 预设价) / 预设价
    • 波动率变化: 20日波动率 vs 当前波动率
    • 成交量异常: 成交量突增/突减
    • 技术指标背离: MACD/RSI 与价格背离
    • 21事件库: 风险事件触发
    
  告警级别:
    • 低级: 价格偏离 > 1%
    • 中级: 价格偏离 > 3% 或 波动率翻倍
    • 高级: 21事件库触发 或 技术指标严重背离
    
  输出:
    • A6情报报告
    • 调整建议
    • 紧急告警
```
