## 1. 系统定位与设计目标

### 1.1 系统定位

Dream Universal Gateway (DUG) 是一个基于 **意图驱动 + 信号驱动** 的智能数字资产交易平台，核心使命是：

> **将人类交易智慧（马丁策略体系、三屏交易理论）编译为可自动化执行的交易系统宪法**
> 
> **核心理念**: 马丁策略经过验证胜率高，结合整体信号系统能更平稳服务用户

### 1.2 核心设计目标

| 目标 | 描述 |
|------|------|
| **智慧化** | 意图识别 + 智能路由，降低用户操作门槛 |
| **系统化** | 马丁策略体系作为底层约束写入系统宪法 |
| **模块化** | 交易流程标准化，信号-执行-监控解耦 |
| **可进化** | A系列流水线持续优化策略，支持自我学习 |
| **双轨制** | 本地模拟盘探索 + 交易系统实盘执行 |

## 2.5 意图识别与智能路由系统 ★ v2.5 新增 · v2.6 边界检查

> **核心定位**: 意图识别+智能路由是系统的"大脑"，根据用户偏好和输入意图，自动路由到最合适的分析路径和执行模式。
>
> **⚠️ 重要约束 (v2.6新增)**: Dream Universal Gateway 是**专业金融数据交易模型**，不是通用AI助手。意图识别层必须执行边界检查，非金融话题必须委婉拒绝。

## 3. 系统宪法：马丁策略体系约束

> **⚠️ 重要声明**: 马丁策略是经过验证有效的交易体系，胜率高，是系统运行的底层约束。任何交易决策必须遵循以下原则。

### 3.1 马丁策略核心原则 (系统宪法)

```yaml
系统宪法 - 马丁策略约束:
  优先级: P0 (不可违背)
  适用场景: 所有交易决策
  
  核心条款:
    1. 多空不对称性原则:
       - 多头方向: 使用多头合约马丁策略
       - 空头方向: 使用空头合约马丁策略
       - 方向选择: 基于 A1-A3 综合判断，多空信号强度决定
       
    2. 分批建仓原则:
       - 建仓触发条件: 周期波动率变化 + 信号触发
       - 信号必须经过评估才能建仓
       - 未达建仓条件不追仓
       
    3. 资金管理原则 (核心):
       - 可用资金分四等份
       - 入场资金: 1份
       - 加仓资金: 最多3次，每次1份
       - 总计: 最多4份资金占用
       - 有效解决"反复 SKIP" 问题
       
    4. 加仓规则:
       - 入场后如出现亏损
       - 在无明确信号情况下，根据系统推荐价位最多加三次仓
       - 加仓间隔基于 20日波动率 + 支撑阻力位
       - 类似于网格交易的分批建仓逻辑
       
    5. 仓位锁定原则:
       - 入仓后，在未平仓前：
         ✅ 不达建仓条件不追仓
         ❌ 禁止手动加仓
         ❌ 禁止提前平仓（除非强制止损）
         
    6. 动态调节原则:
       - 默认止盈止损: 按规则执行
       - 信号强度增强: 可适当提高止盈止损
       - 调节幅度: 受限于最大回撤约束
       
    7. 止盈截断原则:
       - 核心理念: "不赚最后一枚铜板"
       - 执行策略: 移动止盈 + 分批离场
       - 禁止行为: 无限等待更高点位
```

## 4. 三屏交易体系详细设计

### 4.1 三屏交易定位修正

> **⚠️ 核心原则**: 第一屏和第二屏都使用**A1-A3流水线**进行调研分析，区别在于**分析周期**和**输出目标**不同
> 
> **第一屏** = 周线级别 A1-A3 + 技术分析（周线）+ 基本面 → 战略层：**趋势方向 + 双轨策略选择**
> 
> **第二屏** = 日线级别 A1-A3 + 技术分析（日线）+ 支撑阻力位 → 战术层：**预设（入场/加仓/止盈止损）**
> 
> **第三屏** = 小时/分钟级别 A4-A6-A9 → 执行层：**调整 + 实时执行**

| 层级 | 周期 | A1-A3 应用 | 分析重点 | 输出物 |
|------|------|------------|----------|--------|
| **第一屏** | 周线 | ★ 周线级别 A1-A3 | 趋势方向 + 市场情景 + 双轨策略选择 | **策略类型**（合约马丁/现货马丁）+ 方向 |
| **第二屏** | 日线 | ★ 日线级别 A1-A3 | 入场时机预设 + 加仓价位预设 + 止盈止损预设 | **预设价位表**（基于第一屏方向） |
| **第三屏** | 小时/分钟 | A4 + A5 + A6 + A9 | 调整入场/加仓/止盈止损 + 紧急离场 | **执行信号** |

### 4.2 第一屏：周线决策 (A1-A3 周线级) - 方向判断 + 双轨策略选择

> **⚠️ 核心定位**: 第一屏是**战略层**，使用**周线级别A1-A3** + 技术分析（周线）+ 基本面综合调研
> 
> **核心功能**: 确定综合多空方向，选择对应的马丁策略类型（合约马丁 / 现货马丁）
> 
> **执行原则**: 只做方向判断和策略选择，**不预设价格**，**不下单**

```yaml

    - 宏观经济环境 (利率、通胀、美元走势)
    - 行业/赛道基本面 (存储赛道: NAND/SSD供需)

    - 期权 IV 周级别数据

      - 供需矛盾: 周线供应区 vs 周线需求区
      - 多空矛盾: 多头驱动因素 vs 空头驱动因素
      - 宏观矛盾: 宏观经济 vs 行业基本面
    • 输出: 周线矛盾清单 + 矛盾优先级排序
    
  【A2 第一性原理 (周线)】

      - 趋势是沿着阻力最小方向运行
      - 周线趋势具有延续性
      - 识别周线级别的主要矛盾转化点
    • 输出: 周线阻力最小方向 + 趋势动力评级
    
  【A3 沙盘推演 (周线)】
    • 多情景周线推演:

      - 情景B: 空头延续 (概率？%)

    情景B: 空头信号强 (周线评分 ≥ 70)
      → 选择空头策略
      → 可选: 空头合约马丁 / 空头现货马丁
      
    情景C: 多空信号都弱 (周线评分 < 70)
      → 观望等待
      → 等待周线信号明确
      
  ================================================================
  双轨策略选择 (第一屏核心输出)
  ================================================================
  
  【合约马丁策略】
    适用场景:
      • 高波动市场 (周线波动率 > 5%)
      • 趋势明确但波动大 (周线 EMA 排列清晰)
      • 合约杠杆交易
      
    特点:
      • 使用合约（多空双向）
      • 带杠杆（可调整）
      • 收益放大，但风险同步放大

      • 低波动/震荡市场 (周线波动率 < 3%)

      • 稳健增值需求

      • 使用现货（不支持做空）
      • 无杠杆或低杠杆
      • 风险可控，适合稳健策略
      
  策略选择矩阵:

    | 低波动 + 震荡市 | 现货马丁 | 稳健增值 |
```

  ================================================================
  【信号过滤机制】不是所有信号都需要响应 ★ v2.7 新增
  ================================================================

  核心原则: 过滤弱信号，等待强信号

  | 信号强度 | 示例 | 处理方式 |
  |----------|------|----------|
  | **强信号** | 突破关键位、基本面重大变化 | ✓ 执行调整 |
  | **弱信号** | 超时、价格噪音、轻微偏离 | ✗ 忽略 |
  | **需确认信号** | 中等强度信号 | 等更强信号或时间确认 |

  判断标准:

  | 维度 | 强信号标准 | 弱信号标准 |
  |------|-----------|------------|
  | 技术面 | 突破均线/形态确认 | 轻微偏离、噪音 |
  | 基本面 | 宏观数据/政策变化 | 新闻噪音 |
  | 量能 | 放量确认 | 缩量/无量 |

  过滤规则:
    if 信号强度 < 阈值:
        if 无更强技术信号支撑:
            if 无基本面变化:
                → 忽略（等待）
        else:
            → 执行调整

| 层级 | 周期 | A1-A3 应用 | 核心职责 | 输出 |

> - **第一屏是战略层**（周线），决定做什么策略
> - **第二屏是战术层**（日线），决定具体怎么执行（基于战略层方向）
> - **第三屏是执行层**（小时/分钟），实时执行并动态调整

### 7.1.1 L1止盈止损分层执行机制 ★ 核心优化

> **传统马丁策略核心矛盾**: 加仓过程中触发止损，导致永远无法加满
> 
> **DUG解决方案**: 分阶段启用止盈止损，让三层加仓优先完成

```yaml
L1分层执行机制:

  【阶段1: 加仓期（三层未满）】
  ────────────────────────────────────────────────────────────────
  止损策略:
    • 止损仅作为【硬性保护】
    • 阈值放宽至极端情况（跌破Level 3极限价）
    • 目的: 让三层加仓有足够空间完成
    • 触发条件: 价格跌破Level 3价位（极限低价）
    
  止盈策略:
    • 止盈暂不启用
    • 等待三层加满后再启用完整止盈体系
    
  执行原则:
    • 宁可承受更大浮亏，也要完成加仓
    • 三层加满后，均价成本最优
    
  【阶段2: 持仓期（三层加满后）】
  ────────────────────────────────────────────────────────────────
  止损策略:
    • 止损正式启用
    • 止盈止损基于加仓后均价计算
    • 确保盈利空间充足
    
  止盈策略 ★:
    • 常规行情: 止盈保持预设，非趋势加强期不调整
    • 趋势加强期: 可提高止盈空间（扩大止盈目标）
    • 这是相对传统马丁策略的优势
    
  执行原则:
    • 三层加满 = 均价最优 = 止盈止损正式启用
    • 趋势判断 + 止盈动态调整 = 利润最大化
```

### 7.1.2 止盈动态调整规则 ★ v2.8 新增

> **核心优势**: 相对于传统马丁策略的止盈灵活性

```yaml
止盈动态调整矩阵:

  | 趋势状态 | 止盈调整 | 说明 |
  |----------|----------|------|
  | **常规行情** | 不调整 | 保持预设止盈，不频繁变动 |
  | **趋势加强** | 提高止盈 | 扩大止盈目标20-50%，锁定更多利润 |
  | **趋势衰竭** | 提前止盈 | 适当降低止盈，快速离场 |
  | **强信号达标** | 立即止盈 | 不追求最高点，截断利润 |

  趋势加强判定:
    • 价格突破关键阻力位
    • 放量确认趋势
    • 基本面利好持续
    • A6监控确认趋势动能强劲
    
  执行示例:
    预设止盈: $83,000
    趋势加强: 提高至 $85,000 (+2.4%)
    强信号达标: $83,000 立即止盈
```

### 7.2 A9 与马丁策略联动 ★ v2.8 核心修复
