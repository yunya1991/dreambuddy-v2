## 一、策略流程规范（三屏交易体系修正）

### 1.0 核心原则修正

**旧逻辑问题**:
- 第一屏"观望(WAIT)"状态下禁止开仓 → 导致大量时间空仓
- 第二屏信号评分门槛过高 → 有效交易信号被过滤
- 第三屏要求完整A4→A5→A6→A9链路 → 单点阻塞导致整体瘫痪
- 各屏耦合过紧 → 一屏异常则全链路失效

**新核心原则**:
1. **多头和观望都是做多** — 区别仅是趋势强度和仓位大小
2. **第一屏只定方向，不决定是否开仓** — 开仓权归第二屏
3. **第三屏是独立监控层** — 每个SKILL可独立触发，无需完整链路
4. **置信度驱动执行** — 高置信度即刻执行，低置信度SKIP保持原状

### 1.1 第一屏：周线决策（战略层）

**定位**: 确定市场状态和交易方向，不决定是否开仓

**执行频率**: 每周一次（周一开盘前）

**输入**:
- 周线K线数据
- A1深度调研报告
- A2第一性原理分析
- 宏观经济环境

**输出**: 三种市场状态

| 市场状态 | 方向 | 策略类型 | 仓位上限 | 说明 |
|:--------|:-----|:---------|:---------|:-----|
| **强多头** | LONG | 合约马丁 | 60% | EMA多头排列 + MACD金叉 + 放量 |
| **弱多头** | LONG | 现货马丁 | 40% | EMA接近但趋势不明确，偏保守 |
| **观望** | LONG | 现货马丁 | 20% | 无明确方向，默认多头（矛盾论：矛盾永远存在，不做空默认做多） |
| **空头** | SHORT | 合约马丁 | 60% | 明确空头信号 |

**关键修正**:
- ~~观望=禁止开仓~~ → **观望=弱信号多头，20%仓位上限**
- 空头是唯一允许做空的状态
- 策略类型选择考虑：波动率>3%用合约马丁，否则现货马丁
- 需要结合用户风控偏好（保守/积极）

**产物投递**:
- 投递到 6-TRADING 邮箱
- 包含：方向、策略类型、仓位上限、周线评分

### 1.2 第二屏：日线预设（战术层）

**定位**: 在第一屏确定的方向下，进行订单设置（四类订单）

**执行频率**: 每日一次

**前置条件**: 必须有第一屏的方向输出

**核心职责**: 设置四类订单

#### 四类订单

| 订单类型 | 触发条件 | 设置方式 | 说明 |
|:--------|:---------|:---------|:-----|
| **① 入场单** | 机会信号即刻入场 / 无信号按20日波动率挂单 | 强趋势: 市价单；信号一般: 限价挂单 | 马丁策略初始仓位 |
| **② 加仓单** | 根据分析预设挂单 | 限价挂单（基于20日波动率间隔） | 最多3次加仓 |
| **③ 止盈单** | 根据分析预设挂单 | 限价挂单（基于ATR倍数） | 三层加满后启用移动止盈 |
| **④ 止损单** | 亏损超20%强制离场 | 立即计算并挂单 | **硬性约束，不可跳过** |

#### 入场时机决策树

```
第一屏方向确定
    ↓
是否存在高置信度入场信号？
    ├─ YES → 即刻入场（市价单）
    │         仓位 = 基础仓位 × 信号强度比例
    │
    └─ NO → 根据20日波动率计算挂单价
              限价挂单等待触发
```

#### 加仓阶梯规则

```
初始入场: Level 0 (100%基础仓位)
    ↓ 价格下跌 N% (N = 20日波动率 × 0.5, 最小1%)
加仓1: Level 1 (等额加仓)
    ↓ 继续下跌 N%
加仓2: Level 2 (等额加仓)
    ↓ 继续下跌 N%
加仓3: Level 3 (等额加仓, 最后一层)
    → 启用移动止盈保护
```

**关键修正**:
- ~~第二屏评分不够不开仓~~ → **第一屏确定方向后，第二屏必须设置订单**
- 区分"强趋势"和"信号一般"两种模式：
  - **强趋势**（周线评分≥65）: 即刻市价入场 + 预设其他三单
  - **信号一般**（周线评分50-64）: 全部限价挂单，等待触发
- 止损计算：基于入场价，亏损20%即为止损价，**不考虑置信度**

**产物投递**:
- 投递到 6-TRADING 邮箱
- 包含：四类订单详情（价格、数量、触发条件）

### 1.3 第三屏：监控层（执行层）

**定位**: 实时监控，独立信号驱动调仓

**核心修正**:
- ~~A4→A5→A6→A9 完整链路~~ → **每个SKILL独立运行，独立触发**
- 置信度驱动：高置信度执行，低置信度SKIP

#### 独立信号机制

| SKILL | 监控内容 | 触发条件 | 执行动作 |
|:------|:---------|:---------|:---------|
| **A4** 战术验证 | 策略验证 | 发现更优入场/加仓点位 | 置信度≥70: 修改挂单；<70: SKIP |
| **A5** 决策执行 | 综合判断 | 出现执行机会 | 置信度≥70: 执行下单；<70: SKIP |
| **A6** 情报监控 | 市场情报 | P0/P1事件 | 置信度≥80: 紧急调仓；<80: 记录 |
| **A9** 离场决策 | 风险检查 | 止盈/止损/风险事件 | 置信度≥60: 执行离场；<60: SKIP |

#### 置信度规则

```
每个SKILL输出格式:
{
  "signal_type": "add_position | take_profit | stop_loss | emergency",
  "confidence": 0-100,
  "action": "EXECUTE | SKIP | HOLD",
  "reason": "...",
  "order_detail": { ... }
}

执行规则:
  confidence ≥ 阈值 → action = "EXECUTE" → 修改/执行订单
  confidence < 阈值 → action = "SKIP" → 保持原有挂单不变
```

#### 特殊规则

**A9 止损特殊处理**:
- 止损信号无需高置信度 —— 因为第一屏已设置20%硬性止损
- A9 的非高置信度建议一般可以废弃
- 仅"亏损超20%强制离场"是硬性约束

**邮箱投递**:
- 所有执行结果（EXECUTE 或 SKIP）均投递到 6-TRADING 邮箱
- EXECUTE 操作同时触发 → 产物中台 → 前端展示

### 3.1 交易规范SKILL（新增）

**名称**: `dream-trading-workflow-monitor`
**定位**: 6-TRADING 工作流健康监控 + 自愈
**频率**: 每4小时
**投递**: 秘书邮箱

#### 监控维度

| 监控项 | 检查内容 | 异常处理 |
|:-------|:---------|:---------|
| **第一屏健康** | 本周是否已输出方向？方向是否合理？ | 超过周三未输出 → 告警秘书 |
| **第二屏健康** | 今日是否已设置四类订单？ | 超过10:00未设置 → 告警秘书 |
| **订单状态** | 是否有活跃订单？订单是否过期？ | 无活跃订单超3天 → 提醒检查 |
| **第三屏信号** | A4/A5/A6/A9 最近一次运行结果？ | 超过8h无信号 → 告警秘书 |
| **邮箱投递** | 6-TRADING 邮箱是否正常接收？ | 超过2h无新文件 → 告警 |
| **产物中台** | 扫描器是否正常投递？ | 扫描器异常 → 告警秘书 |
| **风控指标** | 当前回撤、仓位占用、保证金率 | 回撤≥15% → 紧急告警秘书 |
| **回测一致性** | 实盘表现 vs 回测偏差 | 偏差超10% → 标记异常 |

#### 自愈能力

```
监控发现异常
    ↓
Level 1: 自动修复（重启自动化、清理过期订单、重新投递）
    ↓ 失败
Level 2: 告警秘书邮箱（附详细诊断报告）
    ↓
Level 3: 等待人工介入
```

