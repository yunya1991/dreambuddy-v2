## 数据源优先级

### Tavily 优先原则

> ✅ **写死经验**: 宏观/情绪/新闻搜索默认使用Tavily

**原因**:
- AI优化搜索，结果结构化
- 延迟低，成功率高
- 支持多语言搜索

**适用场景**:
- 宏观政策变化
- 市场情绪分析
- 新闻事件解读
- 分析师观点收集

### Odaily 补充原则

> ⚠️ **补充工具，非主力**: 链上/ETF数据使用Odaily

**已知问题**:
- SSL超时率约 **30%**
- 响应时间不稳定
- 建议添加重试机制

**适用场景**:
- 链上鲸鱼活动
- ETF资金流向
- 合约持仓数据

**降级策略**:
```
Odaily失败 → Tavily搜索相同关键词 → 标记数据来源变更
```

## 调研方法论

### 正确调研流程

```
1. 明确调研目的
   ├── 记忆调研 → 调用 MEMORY.md / episodes
   ├── 历史调研 → 调用 Archive Center
   ├── 策略调研 → 调用 Strategy Library
   └── 当下调研 → 调用 Tavily → Odaily (fallback)

2. 联网搜索规范
   ├── 必须使用 Tavily 或 Odaily
   ├── 必须标注来源和时间
   ├── 必须记录搜索关键词
   └── 必须评估可信度

3. 证据链构建
   ├── 直接证据: 数值/图表
   ├── 间接证据: 专家观点/媒体报道
   └── 推断证据: 逻辑推演
```

### 评价与取证

```
证据评价维度:
├── 来源可靠性: 官方 > 权威媒体 > 自媒体
├── 时间新鲜度: 最新 > 24h > 7d
├── 逻辑一致性: 多源印证 > 单源
└── 利益相关性: 客观中立 > 利益相关

取证要求:
├── 每条关键信息必须标注来源
├── 外部数据必须标注获取时间
└── 推测内容必须标注"推测"标签
```
