## 情景收敛与背离触发机制 (v1.0 新增 · 2026-04-29)

> **⚠️ 重要升级**: 本机制解决"12个委托混乱"问题
>
> **本质分析**:
> - A4针对A3不同情景推演进行多情景策略设计，**本身没有错**
> - 但多情景委托未经收敛，导致系统混乱
>
> **解决思路**:
> 1. 多情景委托是验证期的合理状态
> 2. 一旦某一情景被验证 → **自动收敛** → 清除其他委托
> 3. 如果验证与A3方向背离 → **触发A6深度调查**

### 1. 情景收敛规则 (Scenario Convergence Rule)

#### 触发条件（必须同时满足）

```

  - 条件1: A4验证情景数 ≥ 1
  - 条件2: 某一情景被验证触发（价格触及/信号确认）
  - 条件3: 该情景概率 ≥ A3主推演情景概率（最大概率情景）
```

## 浮盈蒸发止损机制 (UPL-DECAY-STOPLOSS) 🔴 P0

> **落地来源**: DREAM-PROPOSAL-20260427-001 | rollback_plan_id: ROLLBACK-DREAM-PROPOSAL-20260427-001

> **条件满足**: 阈值设90%(非80%)+30min时间过滤 ✅

### 核心问题

"等反弹"心理疾病 — 浮盈从$90.5蒸发至$0.19(99.8%)，系统HOLD不动导致利润归零。

### 规则定义

```
UPL_DECAY_RULE:
  # 1. 记录浮盈峰值
  当持仓有未实现利润时:
    UPL_peak = max(UPL_peak, current_UPL)

  # 2. 计算衰减率
  UPL_decay_pct = (UPL_peak - current_UPL) / UPL_peak * 100

  # 3. 时间过滤: 衰减需持续>30min
  UPL_decay_start_time: 衰减率首次>50%的时间戳
  if UPL_decay_pct > 50%:
    decay_duration = now - UPL_decay_start_time

  # 4. 分级触发 (初始阈值90%，观察期后可降至80%)
  if UPL_decay_pct >= 50% && decay_duration > 30min:
    → 输出 UPL_DECAY_WARNING (⚠️ 浮盈衰减过半)
  if UPL_decay_pct >= 70% && decay_duration > 30min:
    → 输出 UPL_DECAY_ALERT (🔴 浮盈严重衰减，建议移动TP)
  if UPL_decay_pct >= 90% && decay_duration > 30min:
    → 输出 UPL_DECAY_TRIGGER (🚨 浮盈蒸发>90%，强制锁利)
    → 动作: 移动TP到成本价+最小利润 或 部分平仓50%
```

### 与现有SL关系

- **独立于SL**: UPL-DECAY是"浮盈保护层"，SL是"亏损止损层"，两者正交
- **优先级**: UPL-DECAY先于SL触发（保护利润 > 止损）
- **不替代**: SL@$77,000继续有效，UPL-DECAY是额外保护

- A4每次执行时必须检查所有持仓的UPL衰减状态

```python
        return "DIRECTION_MISMATCH", "同方向实践不足(<2)，方向未验证"
    
    # 检查盈利情况
    profitable_trades = [t for t in same_direction_trades if t["pnl"] > 0]
    if len(profitable_trades) < 2:
        return "UNPROFITABLE", "同方向盈利次数不足(<2)，策略未验证"
    
    # 检查最大亏损
    max_loss = min([t["pnl"] for t in same_direction_trades])
    if max_loss < -5:
        return "HIGH_LOSS", f"单次亏损过大({max_loss}%)，风险未控制"
    
    return "QUALIFIED", f"A4实践合格({len(same_direction_trades)}次同方向，{len(profitable_trades)}次盈利)"
```

```python
def a4_circuit_breaker_check():
    """
    A4亏损熔断检查 - 一旦触发，在A4 SKILL内直接执行完整闭环
    返回: NORMAL / CIRCUIT_BROKEN + 执行结果
    ⚠️ 此函数触发后，A4 SKILL必须按顺序直接落地所有步骤，不得中断
    """
    # 读取A4交易历史
    a4_history = read_a4_trading_history(days=7)
    
    # 计算指标
    single_max_loss = min([t["pnl"] for t in a4_history]) if a4_history else 0
    loss_24h = sum([t["pnl"] for t in a4_history if t["time"] > now() - 24h])
    loss_7d = sum([t["pnl"] for t in a4_history])
    
    # 连续亏损检查
    recent_direction = None
    consecutive_losses = 0
    for t in sorted(a4_history, key=lambda x: x["time"], reverse=True):
        if t["pnl"] < 0:
            if recent_direction is None:
                recent_direction = t["direction"]
                consecutive_losses = 1
            elif t["direction"] == recent_direction:
                consecutive_losses += 1
            else:
                break
        else:
            break
    
    # 熔断判断
    trigger_reason = None
    trigger_value = None
    
    if single_max_loss < -15:
        trigger_reason = "SINGLE_LOSS"
        trigger_value = single_max_loss
    elif loss_24h < -25:
        trigger_reason = "DAILY_LOSS"
        trigger_value = loss_24h
    elif loss_7d < -40:
        trigger_reason = "WEEKLY_LOSS"
        trigger_value = loss_7d
    elif consecutive_losses >= 3:
        trigger_reason = "CONSECUTIVE_LOSS"
        trigger_value = consecutive_losses
    
    if trigger_reason:
        # ⚠️ 触发熔断 - 在A4 SKILL内直接执行完整闭环
        result = execute_circuit_breaker_closure(trigger_reason, trigger_value)
        return "CIRCUIT_BROKEN", result
    
    return "NORMAL", "熔断检查通过"
```

```python
def consult_advisors_internal(evaluation_request):
    """
    【SKILL内直接执行】顾问评估函数
    模拟顾问团评估过程，输出最佳策略推荐
    """
    candidates = evaluation_request["candidates"]
    context = evaluation_request["context"]
    
    # 评分矩阵
    scores = {}
    for candidate in candidates:
        score = 0
        reasons = []
        
        # 1. 战略一致性评分 (权重40%)
        if candidate["direction"] == context["directive"]:
            score += 40
            reasons.append("方向与A3战略完全一致(+40)")
        elif context["directive"] in ["HEDGE", "WAIT"]:
            score += 30
            reasons.append("A3为观望/对冲，策略适配(+30)")
        else:
            score += 10
            reasons.append("方向与A3战略有偏差(+10)")
        
        # 2. 环境适配度评分 (权重30%)
        regime_match = check_regime_match(candidate, context["regime"])
        score += regime_match["score"]
        reasons.append(regime_match["reason"])
        
        # 3. 风险匹配度评分 (权重20%)

        score += risk_match["score"]
        reasons.append(risk_match["reason"])
        
        # 4. 执行可行性评分 (权重10%)

        score += feasibility["score"]
        reasons.append(feasibility["reason"])
        
        scores[candidate["id"]] = {
            "total_score": score,
            "reasons": reasons
        }

    best_id = max(scores, key=lambda x: scores[x]["total_score"])
```

```python
def transfer_best_strategy_to_a5():
    """
    【强制步骤】将A4实践中的最佳策略传递给A5
    在A4每次实践周期结束后自动执行
    """

    strategy_performance = {}
    for practice in a4_practices:
        strategy_id = practice["strategy_id"]
        if strategy_id not in strategy_performance:
            strategy_performance[strategy_id] = {
                "trades": [],
                "total_pnl": 0,
                "win_count": 0,
                "loss_count": 0
            }
        strategy_performance[strategy_id]["trades"].append(practice)
        strategy_performance[strategy_id]["total_pnl"] += practice["pnl"]
        if practice["pnl"] > 0:
            strategy_performance[strategy_id]["win_count"] += 1
        else:
            strategy_performance[strategy_id]["loss_count"] += 1

    # 6. 记录传递日志

    return transfer_doc

def select_best_strategy(performance_data):
    """
    根据表现数据选择最佳策略
    评分维度: 胜率(30%) + 盈亏比(30%) + 夏普比率(20%) + 最大回撤(20%)
    """
    scores = {}
    for strategy_id, data in performance_data.items():
        total_trades = data["win_count"] + data["loss_count"]
        if total_trades < 2:
            continue  # 至少需要2次实践
        
        win_rate = data["win_count"] / total_trades
        avg_profit = calculate_avg_profit(data["trades"])
        avg_loss = calculate_avg_loss(data["trades"])
        profit_factor = abs(avg_profit / avg_loss) if avg_loss != 0 else 999
        sharpe = calculate_sharpe_ratio(data["trades"])
        max_dd = calculate_max_drawdown(data["trades"])
        
        # 综合评分
        score = (
            win_rate * 0.3 +
            min(profit_factor / 3, 1.0) * 0.3 +  # 盈亏比归一化
            min(sharpe / 2, 1.0) * 0.2 +         # 夏普比率归一化
            (1 - abs(max_dd) / 10) * 0.2         # 回撤越小越好
        )
        
        scores[strategy_id] = {
            "score": score,
            "metrics": {
                "win_rate": win_rate,
                "profit_factor": profit_factor,
                "sharpe": sharpe,
                "max_drawdown": max_dd
            }
        }
    
    # 返回最高分策略
    best_id = max(scores, key=lambda x: scores[x]["score"])
    return {
        "strategy_id": best_id,
        "score": scores[best_id]["score"],
        "metrics": scores[best_id]["metrics"]
    }
```

```python
    """
    【强制步骤】基于战略推演联网搜索最佳策略

        if len(strategy_candidates) >= 2:
```

### 4.2 策略搜索代码模板

```python
# 联网搜索最佳策略（在战略战术约束下）
def search_best_strategy(strategy_context):
    """
    根据战略战术约束，联网搜索最佳策略
    """
    # 1. 提取约束条件
    directive = strategy_context["directive_bias"]  # LONG/SHORT/HEDGE/BIAS
    regime = strategy_context["regime"]  # TREND_STRONG/RANGE_BOUND等
    risk_level = strategy_context["risk_level"]  # HIGH/MEDIUM/LOW
    multi_coin = strategy_context.get("multi_coin_signals", {})
    
    # 2. 构建搜索查询
    search_queries = []
    
    # 基于Regime的搜索
    if "TREND" in regime:
        search_queries.append(f"加密货币 趋势跟踪策略 2025年{get_current_month()}")
    elif "RANGE" in regime:
        search_queries.append(f"加密货币 网格交易策略 震荡行情 {get_current_month()}")
    elif "BREAKOUT" in regime:
        search_queries.append(f"加密货币 突破交易策略 2025年{get_current_month()}")
    
    # 基于方向的搜索
    if directive == "HEDGE":
        search_queries.append(f"OKX 多空对冲策略 加密货币 {get_current_month()}")
    elif directive in ["BIAS_LONG", "BIAS_SHORT"]:
        search_queries.append(f"加密货币 仓位偏置策略 对冲 {get_current_month()}")
    
    # 基于工具的搜索
    search_queries.extend([
        f"OKX 合约网格策略 BTC ETH {get_current_month()}",
        f"OKX 马丁格尔策略 最佳参数 {get_current_month()}",
        f"加密货币 动态止盈止损策略 2025年"
    ])
    
    # 3. 执行搜索
    strategies = []
    for query in search_queries:
        results = web_search(query, max_results=3)
        strategies.extend(parse_strategy_results(results))
    
    # 4. 约束过滤
    valid_strategies = []
    for s in strategies:
        # 杠杆约束
        if s["leverage"] > 10:
            continue
        # 风险约束
        if risk_level == "HIGH" and s["risk_score"] > 7:
            continue
        # 方向约束
        if directive == "LONG" and s["direction"] == "SHORT":
            continue
        if directive == "SHORT" and s["direction"] == "LONG":
            continue
        valid_strategies.append(s)
    
    # 5. 排序选择
    valid_strategies.sort(key=lambda x: x["fit_score"], reverse=True)
    return valid_strategies[:3]  # 返回Top3
```

```python
    s_rank = scores_dir / "S_rank.json"
    
    if s_rank.exists():
        with open(s_rank) as f:
            data = json.load(f)
            # 过滤Regime匹配
            practices = [
                e for e in data.get("entries", [])
                if regime in e.get("regime", []) and e.get("tier") in ("P", "S")
```

```python
        完整索引结果 + 最佳策略方案
    """
    result = {
        "timestamp": now(),
        "regime": a3_context["regime"],
        "directive": a3_context["directive"],
        "tier_1_results": {},
        "tier_2_results": {},
        "tier_3_results": {},
        "best_strategy": None,
        "execution_plan": None
    }
    
    # ========== 第一层: 策略索引 ==========
    print("[v7.0] 执行第一层索引: 策略索引...")
    
    # Step 1: 大师策略 (最高优先)
    tier1_step1 = retrieve_master_strategies_v7(a3_context["regime"], top_k=5)
    result["tier_1_results"]["master_strategies"] = tier1_step1
    
    # Step 2: 联网搜索 (Step 1不足时)
    if tier1_step1.get("next_step") == "STEP_2":
        tier1_step2 = search_online_strategies_v7(
            a3_context["regime"], 
            a3_context["directive"]
        )
        result["tier_1_results"]["online_strategies"] = tier1_step2
    
    # Step 3: 实践经验 (兜底)
    tier1_step3 = retrieve_practice_strategies_v7(a3_context["regime"])
    result["tier_1_results"]["practice_strategies"] = tier1_step3
    
    # 合并第一层结果
    all_strategies = merge_strategy_results(
        result["tier_1_results"]
    )
    
    # ========== 第二层: 工具索引 ==========
    print("[v7.0] 执行第二层索引: 工具索引...")
    tier2 = index_trading_tools_v7(
        a3_context["regime"],
        a3_context["directive"]
    )
    result["tier_2_results"] = tier2
    
    # ========== 第三层: 跨资产设计 ==========
    print("[v7.0] 执行第三层索引: 跨资产设计...")
    tier3 = design_cross_asset_v7(a3_context.get("market_context", {}))
    result["tier_3_results"] = tier3
    
    # ========== 生成最佳策略方案 ==========
    print("[v7.0] 生成最佳策略方案...")
    best_strategy = generate_best_strategy_v7(
        strategies=all_strategies,
        tool=tier2,
        cross_asset=tier3,
        a3_context=a3_context
    )
    result["best_strategy"] = best_strategy
    
    # ========== 生成执行计划 ==========
    execution_plan = generate_execution_plan_v7(best_strategy, tier2)
    result["execution_plan"] = execution_plan
    
    return result


def merge_strategy_results(tier1_results: dict) -> list:
    """
    合并三层策略索引结果
    按优先级排序: 大师 > 联网 > 实践
    """
    merged = []
    
    # 1. 添加大师策略 (去重)
    masters = tier1_results.get("master_strategies", {}).get("strategies", [])
    for m in masters:
        merged.append({
            **m,
            "priority": 1,
            "source": "master",
            "score": m.get("score", m.get("source_score", 0))
        })
    
    # 2. 添加联网搜索策略
    online = tier1_results.get("online_strategies", {}).get("results", [])
    for o in online:
        if o["id"] not in [m["id"] for m in merged]:
            merged.append({
                **o,
                "priority": 2,
                "source": "online",
                "score": o.get("relevance", 0.5) * 100
            })
    
    # 3. 添加实践经验策略
    practices = tier1_results.get("practice_strategies", {}).get("practices", [])
    for p in practices:
        if p["id"] not in [m["id"] for m in merged]:
            merged.append({
                **p,
                "priority": 3,
                "source": "practice",
                "score": p.get("score", 0)
            })
    
    # 按优先级 + 评分排序
    merged.sort(key=lambda x: (x["priority"], -x["score"]))
    
    return merged
```

```python
def check_a9_cleanup_flag(inst_id=None, profile="dreamdemo"):
    """检查A9清理标志, 返回标志字典或None"""
    from pathlib import Path
    log_dir = Path("dream-exit-skill-v2/logs")
    flag_files = sorted(log_dir.glob("a9_cleanup_*.json"), reverse=True)
    for f in flag_files:
        import json
        from datetime import datetime
```

### 6.1 置信度评估模型

#### 6.1.1 置信度打分公式

```python
def calculate_confidence_v7(
    si_index: float,      # SI_Index: -100~+100
    edge: float,         # Edge: -100~+100
    regime: str,          # 当前Regime
    execution_regime: str, # 执行时Regime
    price_change: float,  # 价格变化%
    fgi_change: float,     # FGI变化
    funding_change: float,  # 费率变化
    macro_signal: str      # 宏观信号: RISK_ON/OFF/DIVERGE
) -> dict:
    """
    【核心评估函数】计算置信度并决定执行策略
    """
    result = {
        "confidence_score": 0,
        "level": "LOW",  # LOW/MEDIUM/HIGH/CHANGED
        "signal": "HOLD",
        "reason": "",
        "recommendations": []
    }
    
    # ========== 1. 基础分 (基于SI+Edge) ==========
    base_score = 0
    
    # SI_Index贡献 (权重40%)
    if abs(si_index) >= 30:
        base_score += 40 * (abs(si_index) / 100)
    elif abs(si_index) >= 15:
        base_score += 20 * (abs(si_index) / 50)
    else:
        base_score += 5  # 基础分
    
    # Edge贡献 (权重40%)
    if abs(edge) >= 20:
        base_score += 40 * (abs(edge) / 100)
    elif abs(edge) >= 10:
        base_score += 20 * (abs(edge) / 50)
    else:
        base_score += 5
    
    # ========== 2. 方向一致性加分 (权重20%) ==========
    direction_score = 0
    
    # SI与Edge同向加分
    if si_index * edge > 0:  # 同向
        direction_score += 20
        result["reason"] += "SI与Edge同向 +20; "
    
    # Regime一致性
    if regime == execution_regime:
        direction_score += 10
        result["reason"] += "Regime一致 +10; "
    else:
        direction_score -= 10
        result["reason"] += "Regime变化 -10; "
    
    # ========== 3. 市场信号调整 ==========
    signal_adjustment = 0
    
    # 价格变化调整
    if abs(price_change) >= 5:
        signal_adjustment += 5 if price_change > 0 else -5
        result["reason"] += f"价格{'上涨' if price_change > 0 else '下跌'} {price_change:.1f}% "
    
    # FGI变化调整
    if fgi_change >= 10:
        signal_adjustment += 5
        result["reason"] += f"FGI上升 {fgi_change:.0f} +5; "
    elif fgi_change <= -10:
        signal_adjustment -= 5
        result["reason"] += f"FGI下降 {fgi_change:.0f} -5; "
    
    # 宏观信号调整
    if macro_signal == "RISK_OFF":
        signal_adjustment -= 5
        result["reason"] += "宏观避险信号 -5; "
    elif macro_signal == "RISK_ON":
        signal_adjustment += 5
        result["reason"] += "宏观风险偏好 +5; "
    
    # ========== 4. 汇总置信度 ==========
    total_score = base_score + direction_score + signal_adjustment
    result["confidence_score"] = max(0, min(100, total_score))
    
    # ========== 5. 判断置信度级别 ==========
    if result["confidence_score"] >= 70:
        result["level"] = "HIGH"
        result["signal"] = "EXECUTE"  # 触发A5执行
    elif result["confidence_score"] >= 50:
        result["level"] = "MEDIUM"
        result["signal"] = "MONITOR"  # 继续监控
    elif result["confidence_score"] >= 30:
        result["level"] = "LOW"
        result["signal"] = "WAIT"  # 等待更多信号
    else:
        result["level"] = "CHANGED"
        result["signal"] = "UPDATE_A3"  # 触发A3更新
    
    return result
```

```python
def restart_a4_loop_v7(updated_a3: dict, execution_record: dict) -> dict:
```

### 14.4 P0自动补齐代码实现

```python
# P0自动补齐闭环执行器（v7.2新增 - A4 SKILL内闭环）
# 参照C5亏损熔断模式，A4 SKILL内直接执行A1→A2→A3，无需外部触发

_P0_EXECUTED_THIS_CYCLE = False  # 防无限循环：每周期最多执行1次

def a4_p0_auto_replenish(missing_modules: list, reason: str, freshness_hours: float = 0):
    """
    A4 P0自动补齐闭环执行器
    
    当A4检测到A1-A3产物过期/缺失/Regime升级时，
    在A4 SKILL内部直接执行A1→A2→A3补齐，形成完整闭环。
    
    双保险：即使自动化prompt未触发，SKILL内也会自动执行。
    
    Args:
        missing_modules: 缺失模块列表 ["A1","A2","A3"]
        reason: 触发原因 STALE_PRODUCT / PRODUCT_MISSING / REGIME_UPGRADE / DIRECTION_CONFLICT
        freshness_hours: 产物过期时长（仅STALE_PRODUCT时有效）
    """
    global _P0_EXECUTED_THIS_CYCLE
    
    # 防无限循环：每周期最多执行1次
    if _P0_EXECUTED_THIS_CYCLE:
        print("[P0-AUTO] ⚠️ 本周期已执行过P0补齐，跳过防止无限循环")
        return {"status": "SKIP", "reason": "P0_ALREADY_EXECUTED_THIS_CYCLE"}
    
    _P0_EXECUTED_THIS_CYCLE = True
    
    print(f"[P0-AUTO] 🔄 P0自动补齐触发: reason={reason}, missing={missing_modules}, freshness={freshness_hours}h")
    
    # Step 1: 标记P0状态
    a4_status = "P0_REPLENISHING"
    p0_context = {
        "trigger_reason": reason,
        "missing_modules": missing_modules,
        "freshness_hours": freshness_hours,
        "old_regime": extract_regime_from_old_a3(),  # 从旧A3提取
        "timestamp": now()
    }
    
    # 记录P0触发到episode
    write_episode({
        "type": "P0_AUTO_REPLENISH_TRIGGERED",
        "reason": reason,
        "missing": missing_modules,
        "freshness_hours": freshness_hours,
        "a4_status": a4_status
    })
    
    # Step 2: 执行A1深度调研
    if "A1" in missing_modules or reason in ["STALE_PRODUCT", "REGIME_UPGRADE"]:
        print("[P0-AUTO] Step 2: 执行A1深度调研...")
        a1_input = f"P0自动补齐触发 | 原因: {reason}"
        if reason == "REGIME_UPGRADE":
            a1_input += f" | 旧Regime={p0_context['old_regime']}需验证新Regime证据"
        if freshness_hours > 0:
            a1_input += f" | 旧产物{freshness_hours:.0f}h过期"
        
        use_skill("dream-strategy-research", input=a1_input)
        # 产物自动写入: ~/.workbuddy/skills/boss-secretary/reports/trading/a1_research_*.md
        print("[P0-AUTO] ✅ A1完成")
    
    # Step 3: 执行A2第一性原理分析
    if "A2" in missing_modules or reason in ["STALE_PRODUCT", "REGIME_UPGRADE"]:
        print("[P0-AUTO] Step 3: 执行A2第一性原理分析...")
        a2_input = f"基于P0补齐A1新调研 | 触发原因: {reason}"
        
        use_skill("dream-first-principles", input=a2_input)
        # 产物自动写入: ~/.workbuddy/skills/boss-secretary/reports/trading/a2_first_principles_*.md
        print("[P0-AUTO] ✅ A2完成")
    
    # Step 4: 执行A3战略制定
    if "A3" in missing_modules or reason in ["STALE_PRODUCT", "REGIME_UPGRADE", "DIRECTION_CONFLICT"]:
        print("[P0-AUTO] Step 4: 执行A3战略制定...")
        a3_input = f"基于P0补齐A1+A2新分析 | 触发原因: {reason}"
        
        use_skill("dream-strategy-designer", input=a3_input)
        # 产物自动写入: ~/.workbuddy/skills/boss-secretary/reports/trading/a3_strategy_*.md
        print("[P0-AUTO] ✅ A3完成")
        
        # 标记旧A3为SUPERSEDED
        mark_old_a3_superseded(new_a3_reference=latest_a3_path())
    
    # Step 5: 闭环完成
    a4_status = "REPLENISHED"
    print("[P0-AUTO] ✅ P0补齐闭环完成，重新执行A4验证...")
    
    # 记录闭环完成到episode
    write_episode({
        "type": "P0_AUTO_REPLENISH_COMPLETED",
        "reason": reason,
        "new_products": list_new_products(),
        "a4_status": a4_status
    })
    
    return {"status": "REPLENISHED", "a4_status": a4_status}
```

### 14.7 Regime升级检测（P0条件T3特化）

```python
def check_regime_upgrade() -> dict:
    """
    检测A2报告中的Regime与A3战略假设的Regime是否一致
    不一致时触发P0自动补齐（条件T3特化）
    """
    latest_a2 = read_latest("a2_first_principles_*.md")
    latest_a3 = read_latest("a3_strategy_*.md")
    
    if not latest_a2 or not latest_a3:
        return {"upgrade": False, "reason": "MISSING_PRODUCT"}
    
    a2_regime = extract_regime(latest_a2)  # e.g., "TREND_STRONG_UP"
    a3_regime = extract_regime_assumed(latest_a3)  # e.g., "RANGE_BOUND"
    
    # Regime等级映射（用于比较偏差程度）
    regime_levels = {
        "TREND_STRONG_UP": 3,
        "TREND_UP": 2,
        "BIAS_LONG": 1,
        "RANGE_BOUND": 0,
        "BIAS_SHORT": -1,
        "TREND_DOWN": -2,
        "TREND_STRONG_DOWN": -3,
        "FALSE_BREAKOUT_RISK": -4
    }
    
    a2_level = regime_levels.get(a2_regime, 0)
    a3_level = regime_levels.get(a3_regime, 0)
    
    level_diff = abs(a2_level - a3_level)
    
    if level_diff >= 1:
        # Regime偏差≥1级，触发P0自动补齐
        print(f"[P0-AUTO] ⚠️ Regime升级检测: A2={a2_regime}(L{a2_level}) vs A3={a3_regime}(L{a3_level}), 偏差{level_diff}级")
        a4_p0_auto_replenish(
            missing_modules=["A3"],  # A3需要重新制定
            reason="REGIME_UPGRADE",
            freshness_hours=0  # 新鲜度可能OK，但Regime不匹配
        )
        return {"upgrade": True, "a2_regime": a2_regime, "a3_regime": a3_regime, "level_diff": level_diff}
    
    return {"upgrade": False, "a2_regime": a2_regime, "a3_regime": a3_regime, "level_diff": level_diff}
```
