# ⚖️ A0: 矛盾分析理论 — 统一矛盾操作系统 (v1.0)

> **核心定位**: A0不是独立运行的SKILL，而是A1/A2/A3的**内嵌分析操作系统**。
> A1调用时执行"发现矛盾"，A2调用时执行"抓住矛盾"，A3调用时执行"利用矛盾"。
> 独立存在的目的是：统一方法论、后期优化扩展、避免三个SKILL各自发明矛盾处理逻辑。

## 一、哲学基础 — 三源蒸馏

### 1.1 《矛盾论》(毛泽东) — 主框架

| 原理 | 核心要义 | 交易映射 |
|:-----|:---------|:---------|
| **矛盾普遍性** | 矛盾存在于一切事物中，贯穿始终 | 利好利空永远同时存在，**信号不足≠等待** |
| **矛盾特殊性** | 每个矛盾都有独特性，不可套模板 | 每个市场情境的矛盾结构不同，必须具体分析 |
| **主要矛盾** | 复杂事物中必有一个矛盾起主导作用 | 抓住主要矛盾，其他矛盾让路；**只需主要矛盾方向明确即可行动** |
| **矛盾的主要方面** | 矛盾双方中必有一方居于支配地位 | 多方力量>空方力量=做多；量化力量对比，优势方即方向 |
| **矛盾转化** | 矛盾双方在一定条件下相互转化 | 设定转化条件监控点，触发则调整方向 |

### 1.2 《孙子兵法》 — 补充策略维度

| 原理 | 核心要义 | 交易映射 |
|:-----|:---------|:---------|
| **知己知彼** | 了解自己也了解对手 | 量化己方(系统)的偏差+彼方(市场)的信号 |
| **虚实篇** | 避实击虚 | 市场共识过强=实→警惕；被忽视的方向=虚→机会 |
| **势篇** | 顺势而为，不逆势而动 | 主要矛盾方向=势；逆势操作必须有极强证据 |
| **九变篇** | 灵活应变，不可拘泥 | 矛盾转化时必须灵活调整，不可死守旧判断 |

### 1.3 《战争论》(克劳塞维茨) — 补充对抗维度

| 原理 | 核心要义 | 交易映射 |
|:-----|:---------|:---------|
| **摩擦** | 战争中一切都比预期更困难 | 滑点、延迟、情绪干扰=摩擦，必须留安全边际 |
| **重心** | 找到敌方重心并集中攻击 | 主要矛盾=市场重心，所有分析资源集中于主要矛盾 |
| **战争迷雾** | 信息永远不完整 | 接受不确定性，在信息不完整时用矛盾论做出最优判断 |
| **防守优势** | 防守方天然有优势 | 止损就是防守，宁可多设止损也不逆势硬扛 |

## 二、矛盾分类体系 — 7大矛盾维度

> **A1在调研时必须按此分类列举矛盾对，A2在分析时必须按此分类识别主要矛盾**

### 2.1 矛盾分类表

| 编号 | 矛盾维度 | 对立面A (多方) | 对立面B (空方) | 典型场景 |
|:-----|:---------|:--------------|:--------------|:---------|
| **C1** | 资金面矛盾 | ETF净流入、OI增仓 | ETF净流出、OI减仓 | 资金流入但价格不涨 |
| **C2** | 情绪面矛盾 | FGI贪婪、资金费率正 | FGI恐惧、资金费率负 | 情绪极端但资金面中性 |
| **C3** | 技术面矛盾 | EMA多头排列、RSI>50 | EMA空头排列、RSI<50 | 技术信号冲突 |
| **C4** | 宏观矛盾 | 降息/QE、DXY走弱 | 加息/QT、DXY走强 | 宏观利好但市场不买账 |
| **C5** | 地缘矛盾 | 地缘紧张(避险) | 地缘缓和 | 地缘利好但BTC不涨(异常) |
| **C6** | 时序矛盾 | 短期看多(反弹信号) | 长期看空(趋势向下) | 短长期方向冲突 |
| **C7** | 隐性矛盾 | 被压制的信号(做梦产物) | 显性主流信号 | 系统忽视的信号vs共识 |
| **C8** | 宏观矛盾 | 黄金跌/股市涨(风险偏好) | 黄金涨/BTC跌(避险) | 宏观资产与BTC方向背离 |

### 2.2 矛盾评级标准

每个矛盾对需标注以下属性：

```yaml
contradiction_item:
  dimension: "C1"                    # 矛盾维度编号
  side_a: "ETF净流入$19亿"           # 对立面A(多方信号)
  side_b: "OI增仓但价格横盘"         # 对立面B(空方/反向信号)
  strength_a: "HIGH"                 # A方强度: HIGH/MODERATE/LOW
  strength_b: "MODERATE"             # B方强度
  dominance: "A"                     # 当前主导方: A/B/EQUAL
  time_horizon: "中期"               # 时间视角: 短期/中期/长期
  evidence_count_a: 3                # A方证据数量
  evidence_count_b: 2                # B方证据数量
  evidence_quality_a: 0.8            # A方证据质量(0-1)
  evidence_quality_b: 0.6            # B方证据质量(0-1)
  is_primary: false                  # 是否为主要矛盾
  transformation_condition: "ETF连续3日流出" # 转化条件
```

## 三、A0三步法 — 发现→抓住→利用

### Step 1: 发现矛盾 (Discovery) ← A1集成点

> **A1在Phase 3情报整合时，必须调用此步骤**

```
矛盾发现流程:
├── 1.1 遍历8大矛盾维度(C1-C8)
│   └── 对每个维度，列举当前市场中的具体矛盾对
│
├── 1.2 对每个矛盾对标注属性
│   ├── 对立面A和对立面B的具体信号
│   ├── 证据数量(不少于3条)
│   ├── 证据质量(高/中/低)
│   └── 时间视角(短期/中期/长期)
│
├── 1.3 过滤无效矛盾
│   ├── 单方证据<1条 → 降级为"噪声"，不纳入矛盾清单
│   └── 单方强度=LOW且数量=1 → 标记为"次要矛盾"
│
└── 1.4 输出矛盾清单
    └── 格式: contradiction_list (见下方)
```

**输出格式**:
```json
{
  "contradiction_list": [
    {
      "id": "CX_001",
      "dimension": "C1",
      "name": "资金面矛盾",
      "side_a": "ETF连续3日净流入$19亿",
      "side_b": "OI减仓-2.3%，资金费率接近零轴",
      "strength": "MODERATE",
      "dominance": "A",
      "evidence": {
        "a": ["iShares净流入$8.5亿", "Bitwise净流入$5.2亿", "灰度净流入$5.3亿"],

```

**核心规则**:
1. **矛盾普遍性强制**: 必须找到≥2个矛盾对，如果市场看似"方向一致"，找隐性矛盾(C7)
2. **做梦产物必纳入**: C7隐性矛盾必须包含做梦部洞察
3. **禁止"无矛盾"**: 输出"无矛盾"=调研不深入，必须找到被忽视的信号

### Step 2: 抓住矛盾 (Analysis) ← A2集成点

> **A2在Phase 3左右脑辩证时，必须调用此步骤**

```
抓住主要矛盾流程:
├── 2.1 从A1的contradiction_list中筛选主要矛盾
│   ├── 方法: 4维评分法
│   │   ├── 维度1: 力量对比分 = |evidence_count_a×quality_a - count_b×quality_b|
│   │   ├── 维度2: 时间紧迫性分 = 短期>中期>长期 (短期3分/中期2分/长期1分)
│   │   ├── 维度3: 证据一致性分 = 同向信号占比 (0-1)
│   │   └── 维度4: 市场影响权重 = 宏观(0.3)+资金(0.25)+技术(0.2)+情绪(0.15)+其他(0.1)
│   │
│   └── 主要矛盾 = 综合得分最高的矛盾对
│
├── 2.2 识别矛盾的主要方面
│   ├── 方法: dominance_score = side_a_score - side_b_score
│   │   ├── > 0: 多方主导 → 方向UP
│   │   ├── < 0: 空方主导 → 方向DOWN
│   │   └── ≈ 0: 势均力敌 → 取时间紧迫性更高的一方
│   │
│   └── 禁止: dominance≈0时输出"无方向"
│       └── 必须取基本面权重更高的一方(A0§1.3克劳塞维茨:重心原则)
│
├── 2.3 判断矛盾转化可能性
│   ├── 转化条件识别
│   │   ├── C1转化: ETF流向连续反转(3日)
│   │   ├── C2转化: FGI单日变化>20点
│   │   ├── C3转化: EMA金叉/死叉
│   │   ├── C4转化: 央行意外决议/非农数据
│   │   └── C7转化: 被压制信号被主流验证
│   │
│   └── 转化概率评估: LOW(<30%) / MODERATE(30-60%) / HIGH(>60%)
│
└── 2.4 输出主要矛盾分析报告
```

**输出格式**:
```json
{
```

**核心规则**:

3. **必须评估转化**: 主要矛盾会随时间转变，必须设定监控条件

### Step 3: 利用矛盾 (Utilization) ← A3集成点

> **A3在Phase 4战略合成时，必须调用此步骤**

```
利用矛盾驱动战略:
├── 3.1 顺着主要矛盾方向行动（核心原则）

│   ├── 主要矛盾方向=DOWN → 战略倾向做空
│   └── ❌ 禁止: "矛盾未消解=观望"
│       └── ✅ 正确: "矛盾存在=市场有方向，主要矛盾决定方向"
│
├── 3.2 矛盾驱动的仓位调节
│   ├── 主要矛盾清晰(dominance差距>2): 标准仓位
│   ├── 主要矛盾模糊(dominance差距≤2): 半仓
│   ├── 多个矛盾方向一致(≥3个同向): 可加仓至1.5x标准
│   └── 多个矛盾方向冲突: 减仓至0.5x标准，但必须有方向
│
├── 3.3 矛盾转化预判
│   ├── 转化概率HIGH: 准备对冲或止盈

│
├── 3.4 利用次要矛盾
│   ├── 次要矛盾方向与主要矛盾一致: 增强信心
│   ├── 次要矛盾方向与主要矛盾相反: 风险标记
│   └── 次要矛盾突然升级(转化条件触发): 重新评估主要矛盾

└── 3.5 输出矛盾驱动战略
```

```
├── 问题（Question）: ✅ 已覆盖（提出了≥5个矛盾）

├── 推断（Inference）: ⚠️ 需完善（建立完整逻辑链条）
├── 概念（Concepts）: ❌ 需增加（定义"主要矛盾"等概念）
├── 假设（Assumptions）: ❌ 需增加（检查隐含假设）
├── 视角（Viewpoints）: ❌ 需增加（考虑对立视角）
└── 意涵（Implications）: ⚠️ 需增加（分析决策后果）
```

**强制要求**：
- 覆盖≥6个元素（≥75%）才能输出
- 未覆盖的元素，必须在报告中说明原因

#### 步骤B：认知偏差识别（Kahneman的7大偏差）

在**Step 2: 抓住矛盾**之后，增加检查：

```
认知偏差识别清单：
├── 锚定效应：是否锚定在某一价格/数据？
├── 框架效应：是否只考虑单一框架？
├── 可用性偏差：是否依赖近期数据？
├── 过度自信：是否高估了预测准确率？
├── 损失厌恶：是否因恐惧损失而不敢行动？
├── 确认偏误：是否只关注支持自己观点的证据？
└── 后见之明偏差：是否觉得自己"早就知道"？
```

**强制要求**：
- 识别≥4个偏差（≥60%）才能输出
- 对每个识别出的偏差，必须说明"如何纠正"

### 7.3 A8批评闭环机制

```
A0输出 → A8批评 → A0改进 → A8再批评 → 通过（PASS）
```

**闭环规则**：
1. A0每次输出后，必须调用A8进行批评
2. A8的批评报告必须包含在A0的输出目录中
3. A0根据A8的批评进行改进，并在下次输出中体现
4. A8再次批评，确认改进效果
5. 循环直到A8评为PASS（所有检查项通过）

**文件命名规则**：
- A0输出：`contradiction_map_YYYYMMDD.json`
