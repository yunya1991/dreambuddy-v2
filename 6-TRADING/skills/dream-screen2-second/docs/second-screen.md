## 二、信号强度评估与资金管理 ★★★

### 2.1 信号强度评估

```yaml
信号强度评估:

  强信号: 日线评分 ≥ 70 + 与周线共振
    → 全仓入场（100%仓位）
    → 不等待回落
    → 市价单立即入场
    
  中信号: 日线评分 50-69 或 与周线部分共振
    → 半仓入场（60%仓位）
    → 可等待小额回落
    → 限价单分批入场
    
  弱信号: 日线评分 < 50 或 与周线背离
    → 轻仓试探（30%仓位）★
    → ❌ 不再"等待回落" → 可能错过趋势
    → ✅ 改为"轻仓试探" → 默认多头
    → 限价单试探
    
  无明确信号: 默认多头（除非周线明确看空）
    → 最小仓位试探（10%仓位）★
    → 保持市场存在感
    → 限价单最小试探
```

### 2.2 资金管理规则 ★ 核心创新

```yaml
资金管理规则:

  仓位计算公式:
    实际仓位 = 基础仓位 × (信号强度 / 100) × 风险系数
    
  其中:
    • 基础仓位: 根据策略类型确定
      - 合约马丁: 20%
      - 现货马丁: 25%
        
    • 信号强度: 0-100 评分
      - 日线评分 + 周线共振加成
      - 强信号(≥70): 100%
      - 中信号(50-69): 60%
      - 弱信号(<50): 30%
      - 无信号: 10%
        
    • 风险系数: 0.5-1.5
      - 根据波动率调整
      - 波动率 > 5%: 风险系数 = 0.5-0.8
      - 波动率 3-5%: 风险系数 = 0.8-1.2
      - 波动率 < 3%: 风险系数 = 1.2-1.5
        
    • 最大回撤约束: ≤ 20%
      - 实时计算当前回撤
      - 回撤 ≥ 15%: 触发警告，暂停加仓
      - 回撤 ≥ 20%: 强制止损，全部平仓

  示例计算:
    情景1: 强信号 + 合约马丁 + 波动率4%
      - 实际仓位 = 20% × (100/100) × 1.0 = 20%
        
    情景2: 中信号 + 现货马丁 + 波动率2%
      - 实际仓位 = 25% × (60/100) × 1.4 = 21%
        
    情景3: 弱信号 + 合约马丁 + 波动率6%
      - 实际仓位 = 20% × (30/100) × 0.6 = 3.6%
        
    情景4: 无信号 + 现货马丁 + 波动率3%
      - 实际仓位 = 25% × (10/100) × 1.2 = 3%
```

### 3.2 A1 矛盾调查 (日线级别)

```yaml
A1 矛盾调查 (日线):

  在周线方向的框架下，识别日线级别矛盾:
    
    C1: 日线供需矛盾
      - 日内支撑位 vs 日内阻力位
      - 示例: BTC 日线支撑 $78,000 vs 阻力 $82,000
        
    C2: 与周线方向的一致性检验
      - 日线是否与周线共振
      - 示例: 周线多头，日线回调 = 健康回调
        
    C3: 日内驱动因素分析
      - 消息面 vs 技术面
      - 示例: 消息面利好但技术面超买
        
    C4: 成交量矛盾
      - 价格上涨但成交量萎缩
      - 示例: 价量背离
        
    C5: 技术指标矛盾
      - 日线MACD vs RSI 背离
      - 示例: MACD金叉但RSI超买
        
    C6: 时间周期矛盾
      - 日线与小时线背离
      - 示例: 日线多头但小时线空头
        
    C7: 隐性矛盾
      - 未被市场注意的关键矛盾
      - 示例: 衍生品未平仓合约异常

  输出:
    - 日线矛盾清单
    - 与周线方向的共振/背离判断
    - 矛盾优先级排序
```

### 3.3 A2 第一性原理 (日线级别)

```yaml
A2 第一性原理 (日线):

  核心原理1: 市场总是沿着阻力最小方向运行
    • 分析日线趋势动力
    • 日内阻力最小方向
    • 日内关键突破点识别
    
  核心原理2: 趋势具有延续性
    • 是否与周线方向共振
    • 日线趋势延续概率
    
  日线阻力最小方向判定:
    • 多头阻力最小:
      - 日线EMA多头排列
      - MACD日线金叉
      - 成交量配合
      - 与周线方向共振
        
    • 空头阻力最小:
      - 日线EMA空头排列
      - MACD日线死叉
      - 成交量萎缩
      - 与周线方向共振
        
    • 震荡 (阻力平衡):
      - 日线EMA纠缠
      - MACD日线横盘
      - 成交量低迷
      - 与周线方向背离

  输出:
    - 日线阻力方向: [多/空/震荡]
    - 关键价位: [$XX,XXX]
    - 与周线一致性评级: [强共振/部分共振/背离]
```

## 四、三大预设详细设计

### 4.1 预设1: 入场时机预设 ★ v3.0 核心修改

```yaml
预设1: 入场时机预设

  核心原则: 弱信号不下漏趋势 + 默认多头 + 强度值资金控制
    
  信号强度评估:
    强信号: 日线评分 ≥ 70 + 与周线共振 → 全仓入场
    中信号: 日线评分 50-69 或 与周线部分共振 → 半仓入场
    弱信号: 日线评分 < 50 或 与周线背离 → 轻仓试探（默认多头）
    无明确信号: 默认多头（除非周线明确看空） → 最小仓位试探
    
  资金管理规则:
    实际仓位 = 基础仓位 × (信号强度 / 100) × 风险系数
    
  入场决策矩阵:
    | 信号强度 | 默认方向 | 仓位比例 | 入场方式 | 说明 |
    |----------|----------|----------|----------|------|
    | 强信号 (≥70) | 跟随周线 | 100% | 市价单立即入场 | 不等待回落 |
    | 中信号 (50-69) | 跟随周线 | 60% | 限价单分批入场 | 可等待小额回落 |
    | 弱信号 (<50) | 默认多头 | 30% | 限价单试探 | 不下漏趋势 ★ |
    | 无信号 | 默认多头 | 10% | 限价单最小试探 | 保持市场存在感 ★ |
    
  情景1 (弱信号/无信号):
    • ❌ 不再"等待回落" → 可能错过趋势
    • ✅ 改为"轻仓试探" → 默认多头 + 最小仓位
    • 示例: 弱信号 → 30%仓位 @ 当前价 ± 1%
    • 示例: 无信号 → 10%仓位 @ 当前价 ± 0.5%
    
  情景2 (强信号/中信号):
    • 强信号 → 全仓入场（100%仓位）
    • 中信号 → 半仓入场（60%仓位）
    • 可适当提前入场，不等待回落
    • 预设入场价格: 当前价 ± 1%
    
  ★ 关键改进:
    - 弱信号不再"等待回落"，而是"轻仓试探"
    - 无明确信号时"默认多头"（除非周线明确看空）
    - 仓位大小由"信号强度值"动态调整
    
  ★ 预设完成 → 直接在交易所挂限价单
```

### 4.2 预设2: 加仓阶梯预设 ★★ 核心原则

```yaml
预设2: 加仓阶梯预设

  核心原则:
    ──────────────────────────────────────────────────────────────
    原则1: 最多3次加仓（4层仓位：初始 + 3次加仓）
    原则2: 必须分阶梯预设（每层阶梯有明确价格间隔）
    原则3: 阶梯基于波动率等指标计算（常规情景）
    原则4: 监控出重要机遇则动态调整（但必须保持阶梯结构）
    ──────────────────────────────────────────────────────────────
    
  加仓阶梯间隔计算:
    阶梯间隔 = max(波动率 × k, 最小间隔)
    示例: BTC $80,000 × 3%波动率 × 0.5k = $1,200间隔
    
  加仓阶梯示意（做多情景）:
    Level 0: $79,800 (入场, 实际仓位%)
    Level 1: $79,000 (间隔 $800, 实际仓位%)
    Level 2: $78,200 (间隔 $800, 实际仓位%)
    Level 3: $77,500 (间隔 $700, 极限阶梯, 实际仓位%)
    
  仓位分配（基于信号强度）:
    强信号:
      Level 0: 50%
      Level 1: 30%
      Level 2: 20%
      Level 3: 0% (不加仓)
        
    中信号:
      Level 0: 60%
      Level 1: 25%
      Level 2: 15%
      Level 3: 0% (不加仓)
        
    弱信号:
      Level 0: 100% (仅入场，不加仓)
      Level 1-3: 0%
        
    无信号:
      Level 0: 100% (仅试探性入场)

  动态调整规则 ★ v2.8 核心进化:
    ──────────────────────────────────────────────────────────────
    核心原则: 基于价值评估做调整，追求更优价格
    
    ┌─────────────────────────────────────────────────────────┐

    ├─────────────────────────────────────────────────────────┤

    │  A5 价值入场  │  现在入场吗?      │  信号强度 + 时机判断    │

      • A4 信号强度 → 影响调整幅度
      
    A5 价值评估 → 入场决策 ★★★☆☆:
      • A5 强信号 → 立即执行阶梯调整
      • A5 弱信号 → 等待A4确认后再调整
      
    A6 价值评估 → 时机监控 ★★★☆☆:
      • A6 发现更优价格 → 触发阶梯动态调整
      • A6 发现风险信号 → 触发A9风险评估
      
    A9 风险评估 → 离场决策 ★★★★★:
      • A9 强制离场: 基本面恶化 / 21事件触发
```

### 4.3 预设3: 止盈止损预设

```yaml
预设3: 止盈止损预设

  弱信号情景:
    • 止损: 基于20日波动率和支撑阻力位
    • 止盈: 基于阻力位和波动率
    • 最多加仓3次，资金分四等份
    • 最大回撤约束: ≤ 20%
      
  强信号情景:
    • 调整入场时机、加仓位、止盈位和止损位
    • 信号强度影响调整幅度
    • 遵循系统宪法约束
    • 最大回撤约束: ≤ 20%
    
  ★ v2.8 重要更新: A9 L1止损分层执行
    ──────────────────────────────────────────────────────────────
    加仓期 (Level 0-2):
      • 仅硬性保护止损（防止爆仓）
      • 不止盈（给市场波动空间）
      • 止损位: 基于支撑位和波动率
      • 最大回撤: ≤ 20%
        
    三层加满后 (Level 3):
      • 启用完整止盈止损
      • 移动止盈: 盈利超10% → 移动止盈到成本价
      • 分批离场: 盈利超20%/30%/50% → 分别离场30%/30%/40%
    ──────────────────────────────────────────────────────────────
    
  止损计算方法:
    止损位 = 入场价 × (1 - 止损百分比)
    
    其中:
      • 弱信号: 止损百分比 = 2 × 20日波动率
      • 强信号: 止损百分比 = 1.5 × 20日波动率
      • 最大止损: 不超过20%（最大回撤约束）
        
  止盈计算方法:
    止盈位 = 入场价 × (1 + 止盈百分比)
    
    其中:
      • 弱信号: 止盈百分比 = 3 × 20日波动率
      • 强信号: 止盈百分比 = 5 × 20日波动率
      • 移动止盈: 盈利超10% → 移动止盈到成本价
        
  最大回撤监控:
    • 实时计算当前回撤 = (当前权益 - 历史最高权益) / 历史最高权益
    • 回撤 ≥ 15%: 触发警告，暂停加仓
    • 回撤 ≥ 20%: 强制止损，全部平仓
```

## 七、第二屏风险控制

### 7.1 最大回撤约束 ★ 核心新增

```yaml
最大回撤约束:

  第二屏职责:
    • 明确最大回撤约束: ≤ 20%
    • 将最大回撤约束传递到第三屏
    • 根据信号强度调整仓位（弱信号小仓位）
    
  回撤计算公式:
    当前回撤 = (当前权益 - 历史最高权益) / 历史最高权益 × 100%
    
  应对措施:
    • 回撤 15-20%: 触发警告，暂停加仓
    • 回撤 ≥ 20%: 强制止损，全部平仓
    
  第二屏预防机制:
    • 弱信号: 30%仓位试探（降低风险）
    • 无信号: 10%仓位试探（最小化风险）
    • 加仓阶梯: 严格遵循阶梯间隔（不随意加仓）
```

### 7.2 资金使用限制

```yaml
资金使用限制:

  单层级仓位限制:
    • 单层级仓位 ≤ 账户20%
    • 累计马丁仓位 ≤ 账户60%
    • 预留至少40%作为安全垫
    
  仓位计算公式:
    实际仓位 = 基础仓位 × (信号强度 / 100) × 风险系数
    
  示例:
    强信号 + 合约马丁 + 波动率4%:
      - 实际仓位 = 20% × (100/100) × 1.0 = 20% ✅
        
    弱信号 + 合约马丁 + 波动率6%:
      - 实际仓位 = 20% × (30/100) × 0.6 = 3.6% ✅
      - 符合"弱信号轻仓试探"原则
```
