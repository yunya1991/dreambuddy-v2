# 第二屏: 日线预设 SKILL (dream-screen2-second)

> **SKILL 定位**: 三屏交易体系的**战术层**
>
> **核心职责**: 基于第一屏方向，输出三大预设（入场/加仓/止盈止损），制定可挂单计划
>
> **执行周期**: 每日 09:00 (或手动触发)
>
> **版本**: v1.1 | 最后更新: 2026-05-16

---

### 1.2 核心原则

| 原则 | 说明 |
|------|------|
| ✅ 继承方向 | 不独立判断方向，必须遵循第一屏 |
| ✅ 输出可挂单 | 预设完成即可在交易所挂限价单 |
| ❌ 不执行交易 | 执行留给第三屏 |
| ★ 弱信号轻仓 | 不等待回落，轻仓试探 |
| ★ 默认多头 | 无明确信号时默认多头 |

### 4.2 贝叶斯参数优化 ★ 新增

```yaml
贝叶斯参数优化流程:

  ★ 调用: dream-bayesian-opt SKILL

  ──────────────────────────────────────────────
  【优化目标】: 找到最优的加仓间隔和止盈止损比例
  
  参数空间定义:
    optimization_space:
      # 加仓相关
      entry_signal_threshold: [50, 80]    # 入场信号阈值
      martingale_levels: [2, 4]            # 加仓层数
      level_spacing_pct: [1.0, 5.0]        # 阶梯间隔 (%)
      level_spacing_k: [0.3, 0.8]          # 波动率系数k
      
      # 止损相关
      stop_loss_pct: [10, 25]              # 止损百分比
      stop_loss_atr_mult: [1.5, 3.0]       # 止损ATR倍数
      
      # 止盈相关  
      take_profit_1x: [2.0, 4.0]           # 第一档止盈ATR倍数
      take_profit_2x: [3.0, 6.0]           # 第二档止盈ATR倍数
      take_profit_3x: [5.0, 8.0]           # 第三档止盈ATR倍数
      
      # 仓位相关
      weak_position_pct: [10, 40]          # 弱信号仓位
      strong_position_pct: [60, 100]       # 强信号仓位

  贝叶斯优化流程:
    Step 1: 先验分布设定
      - 基于当前策略参数作为先验均值
      - 设定合理的变化范围 (如±30%)
      
    Step 2: 似然函数计算
      - 对每组参数运行快速回测 (过去60个交易日)
      - 计算该参数组合的收益分布 P(数据|参数)
      
    Step 3: 后验分布更新
      - P(参数|数据) ∝ P(数据|参数) × P(参数)
      - 选择后验均值或MAP估计作为推荐参数
      
    Step 4: 敏感性分析
      - 识别哪些参数对结果最敏感
      - 对高敏感参数给出更保守的建议

  【输出格式】
    bayesian_optimization_result:
      recommended_params:
        level_spacing: $XXX (原值 ± Δ%)
        stop_loss: X% (原值 ± Δ%)
        take_profit_levels: [Xx, Xx, Xx] ATR
        
      confidence_interval_95:
        level_spacing: [$min, $max]
        
      sensitivity_ranking:
        1. stop_loss_pct (影响度: 高)
        2. level_spacing_k (影响度: 中)
        3. take_profit_3x (影响度: 低)
        
      recommendation: 
        "采用/调整/拒绝" 当前参数集
      reasoning:
        "基于N次回测采样，优化后预期收益提升X%，最大回撤降低Y%"
  ──────────────────────────────────────────────
```

## 五、Phase 4: 三大预设输出

### 5.1 信号强度评估

```yaml
信号强度评估:

  强信号: 日线评分 ≥ 70 + 与周线共振
    → 全仓入场（100%基础仓位）
    → 不等待回落
    
  中信号: 日线评分 50-69 或 与周线部分共振
    → 半仓入场（60%基础仓位）
    → 可等待小额回落
    
  弱信号: 日线评分 < 50 或 与周线背离
    → 轻仓试探（30%基础仓位）★
    → 不再"等待回落"，改为"轻仓试探"
    
  无明确信号: 默认多头（除非周线明确看空）
    → 最小仓位试探（10%基础仓位）★
    → 保持市场存在感
```

### 5.2 资金管理公式

```yaml
资金管理规则 ★ 核心公式:

  实际仓位 = 基础仓位 × (信号强度 / 100) × 风险系数 × 回测修正系数

  其中:
    • 基础仓位: 合约马丁 20% / 现货马丁 25%
    • 信号强度: 0-100 评分
    • 风险系数: 0.5-1.5 (根据波动率调整)
    
    • ★ 回测修正系数 (新增): 
      - 回测通过: 1.0 (不变)
      - 回测警告: 0.7 (降级)
      - 回测优秀(Sharpe>2): 1.1 (可小幅提升，但不超过原值120%)

  最大回撤约束:
    • 单层级仓位 ≤ 账户 20%
    • 累计马丁仓位 ≤ 账户 60%
    • 预留至少 40% 作为安全垫
```

### 5.3 预设1: 入场时机

```yaml
预设1: 入场时机:

  入场决策矩阵:
    | 信号强度 | 默认方向 | 基础仓位% | 入场方式 |
    |----------|----------|-----------|----------|
    | 强信号≥70 | 跟随周线 | 100% | 市价单立即入场 |
    | 中信号50-69| 跟随周线 | 60% | 限价单分批入场 |

    偏移量基于历史相似行情的最优入场偏差统计
```

### 5.4 预设2: 加仓阶梯

```yaml
预设2: 加仓阶梯:

  核心原则:
    • 最多3次加仓（4层仓位：Level 0-3）
    • 分阶梯预设（每层有明确价格间隔）
    • 阶梯间隔 = max(波动率 × k, 最小间隔)
    • ★ k值由贝叶斯优化确定

  加仓示意 (做多):
```
