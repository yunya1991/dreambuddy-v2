## 三、A5 决策执行详细设计

### 3.1 A5 核心功能

```yaml
A5 决策执行:

  功能:
    • 根据A4验证报告 + A6情报，综合判断并执行
    • A5 = A4验证报告 + A6情报 + 综合判断 → 执行下单
    
  决策链路:
    A4验证报告 → A6情报 → 综合判断 → 执行下单
    
  综合判断因素:
    • A4验证结果 (PASS/FAIL)
    • A6情报 (正常/异常/风险)
    • 第二屏预设 (入场/加仓/止盈止损)
    • 最大回撤约束 (≤20%)
    
  输出:
    • 交易执行结果
    • 执行确认
    • 实际仓位 / 信号强度调整
    
  价值: ★★★☆☆ (发出入场指令)
```

### 3.2 A5 执行流程图

```
A5决策执行流程:

  输入:
    • A4验证报告 (PASS/FAIL)
    • A6情报 (正常/异常/风险)
    • 第二屏预设 (入场/加仓/止盈止损)
    • 当前仓位信息
    
  处理:
    ──────────────────────────────────────────────────────────────
    【步骤1: A4验证检查】
      • A4 PASS → 继续
      • A4 FAIL → 延迟执行，等待A4重新验证
        
    【步骤2: A6情报检查】
      • A6 正常 → 继续
      • A6 异常 → 谨慎执行，降低仓位
      • A6 风险 → 暂停执行，触发A9风险评估
        
    【步骤3: 最大回撤检查】★
      • 回撤 < 15% → 正常执行
      • 回撤 15-20% → 警告，暂停加仓
      • 回撤 ≥ 20% → 强制止损，全部平仓
        
    【步骤4: 综合判断】
      • 基于A4+A6+最大回撤+第二屏预设
      • 调整实际仓位 (根据信号强度)
      • 决定执行/延迟执行/不执行
        
    【步骤5: 执行下单】
      • 市价单 或 限价单
      • 记录执行结果
      • 更新仓位信息
    ──────────────────────────────────────────────────────────────
    
  输出:
    • 执行结果: [成功/失败/延迟]
    • 执行类型: [入场/加仓/止盈/止损]
    • 执行价格: [$XX,XXX]
    • 执行数量: [?] (根据实际仓位计算)
    • 当前回撤: [%] (必须 ≤20%)
```

### 5.2 A9 L1 止损分层执行详细设计

```yaml
A9 L1 止损分层执行:

  加仓期 (Level 0-2):
    ──────────────────────────────────────────────────────────────
    止损策略:
      • 仅硬性保护止损（防止爆仓）
      • 不止盈（给市场波动空间）
      
    止损位计算:
      止损位 = 入场价 × (1 - 止损百分比)
      
      其中:
        • 弱信号: 止损百分比 = 2 × 20日波动率
        • 强信号: 止损百分比 = 1.5 × 20日波动率
        • 最大止损: 不超过20%（最大回撤约束）
        
    示例 (BTC/USDT, 多头):
      • 入场价: $79,800
      • 20日波动率: 4.2%
      • 弱信号止损百分比: 2 × 4.2% = 8.4%
      • 止损位: $79,800 × (1 - 8.4%) = $73,104
    ──────────────────────────────────────────────────────────────
    
  三层加满后 (Level 3):
    ──────────────────────────────────────────────────────────────
    止损策略:
      • 启用完整止盈止损
      • 移动止盈: 盈利超10% → 移动止盈到成本价
      • 分批离场: 盈利超20%/30%/50% → 分别离场30%/30%/40%
      
    移动止盈规则:
      • 盈利超10%:
          - 移动止盈到成本价
          - 确保至少不亏损
          
      • 盈利超20%:
          - 离场30%仓位（锁定利润）
          - 剩余70%继续持有
          
      • 盈利超30%:
          - 再离场30%仓位
          - 剩余40%继续持有
          
      • 盈利超50%:
          - 离场剩余40%仓位
          - 全部平仓
          
    最大回撤监控:
      • 实时计算当前回撤 = (当前权益 - 历史最高权益) / 历史最高权益
      • 回撤 ≥ 15%: 触发警告，暂停加仓
      • 回撤 ≥ 20%: 强制止损，全部平仓
    ──────────────────────────────────────────────────────────────
```

## 八、第三屏风险控制

### 8.1 最大回撤约束 ★ 核心新增

```yaml
最大回撤约束:

  第三屏职责:
    • 实时监控当前回撤
    • 回撤 ≥ 15%: 触发警告，暂停加仓
    • 回撤 ≥ 20%: 强制止损，全部平仓
    
  回撤计算公式:
    当前回撤 = (当前权益 - 历史最高权益) / 历史最高权益 × 100%
    
    其中:
      • 当前权益 = 当前仓位市值 + 可用资金
      • 历史最高权益 = 历史上权益的最高值
      
  应对措施:
    • 回撤 < 15%: 正常执行
    • 回撤 15-20%: 触发警告，暂停加仓，仅执行止损
    • 回撤 ≥ 20%: 强制止损，全部平仓，等待下一次机会
```

### 8.2 21事件库

```yaml
21事件库:

  事件类型:
    • 黑天鹅事件 (如: 交易所暴雷、极端行情)
    • 监管政策突变 (如: 禁止加密货币交易)
    • 交易所异常 (如: 无法登录、无法交易)
    • 流动性危机 (如: 无法平仓、滑点巨大)
    
  事件严重程度:
    • 高级: 必须立即离场
    • 中级: 谨慎执行，降低仓位
    • 低级: 注意观察，可以执行
    
  事件库维护:
    • 每次触发后记录到MEMORY.md
    • 每周回顾事件库
    • 根据A8验证结果调整事件库
```
