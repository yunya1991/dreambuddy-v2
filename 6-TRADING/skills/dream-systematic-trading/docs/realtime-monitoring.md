## 一、实时监控系统核心定位

### 1.1 系统定义

**实时监控系统 = A6情报监控 + A9离场决策**

```
┌─────────────────────────────────────────────────────────────────────┐
│                    实时监控系统                                      │
├─────────────────────────────────────────────────────────────────────┤
│                                                                   │
│  核心功能:                                                         │
│  1. A6情报监控 (实时监控)                                        │
│  2. A9离场决策 (四层离场)                                        │
│  3. 最大回撤监控 ★ 核心新增                                     │
│                                                                   │
│  监控频率:                                                         │
│  • 正常状态: 每小时运行                                          │
│  • 异常状态: 每15分钟运行                                        │
│  • 紧急状态: 实时监控                                            │
│  • 最大回撤监控: 实时计算                                        │
│                                                                   │
└─────────────────────────────────────────────────────────────────────┘
```

### 1.2 监控维度

| 监控维度 | 指标 | 告警级别 |
|:----------|:-----|:---------|
| **价格偏离度** | (当前价 - 预设价) / 预设价 | >1%低级 / >3%中级 |
| **波动率变化** | 20日波动率 vs 当前波动率 | 翻倍=中级告警 |
| **成交量异常** | 成交量突增/突减 | 突增=中级告警 |
| **技术指标背离** | MACD/RSI 与价格背离 | 严重背离=高级告警 |
| **21事件库** | 风险事件触发 | 触发=高级告警 |
| **最大回撤** ★ | (当前权益 - 历史最高权益) / 历史最高权益 | ≥15%警告 / ≥20%强制止损 |

## 三、A9 离场决策详细设计

### 3.1 A9 四层离场决策链

```yaml
A9 离场决策:

  四层离场决策链:
    
    L1: 技术止盈止损 ★ v2.8 分层执行
      • 加仓期 (Level 0-2):
          - 仅硬性保护止损（防止爆仓）
          - 不止盈（给市场波动空间）
          - 止损位: 基于支撑位和波动率
          - 最大回撤约束: ≤ 20%
          
      • 三层加满后 (Level 3):
          - 启用完整止盈止损
          - 移动止盈: 盈利超10% → 移动止盈到成本价
          - 分批离场: 盈利超20%/30%/50% → 分别离场30%/30%/40%
          
    L2: 信号反转
      • A4/A5信号反转
      • 日线与周线严重背离
      • 趋势动力消失
      
    L3: 风险事件触发 (21事件库)
      • 黑天鹅事件
      • 监管政策突变
      • 交易所异常
      • 流动性危机
      
    L4: 参数优化
      • 止盈止损动态调整
      • 加仓阶梯优化
      • 仓位管理优化
      
  离场执行:
    • 强制离场: L3风险事件触发
    • 条件离场: L1/L2信号触发
    • 分批离场: 盈利超20%/30%/50% → 分别离场30%/30%/40%
    
  最大回撤监控: ★ 核心新增
    • 实时计算当前回撤
    • 回撤 ≥ 15%: 触发警告，暂停加仓
    • 回撤 ≥ 20%: 强制止损，全部平仓
    
  输出:
    • 离场执行确认
    • MEMORY日志记录
    • A9离场报告
    
  价值: ★★★★★ (止盈止损 + 紧急离场)
```

### 3.2 A9 L1 止损分层执行详细设计

```yaml
A9 L1 止损分层执行:

  加仓期 (Level 0-2):
    ──────────────────────────────────────────────────────────────
    止损策略:
      • 仅硬性保护止损（防止爆仓）
      • 不止盈（给市场波动空间）
      
    止损位计算:
      止损位 = 入场价 × (1 - 止损百分比)
      
      其中:
        • 弱信号: 止损百分比 = 2 × 20日波动率
        • 强信号: 止损百分比 = 1.5 × 20日波动率
        • 最大止损: 不超过20%（最大回撤约束）
        
    示例 (BTC/USDT, 多头):
      • 入场价: $79,800
      • 20日波动率: 4.2%
      • 弱信号止损百分比: 2 × 4.2% = 8.4%
      • 止损位: $79,800 × (1 - 8.4%) = $73,104
      
    最大回撤监控:
      • 实时计算当前回撤
      • 回撤 ≥ 15%: 触发警告，暂停加仓
      • 回撤 ≥ 20%: 强制止损，全部平仓
    ──────────────────────────────────────────────────────────────
    
  三层加满后 (Level 3):
    ──────────────────────────────────────────────────────────────
    止损策略:
      • 启用完整止盈止损
      • 移动止盈: 盈利超10% → 移动止盈到成本价
      • 分批离场: 盈利超20%/30%/50% → 分别离场30%/30%/40%
      
    移动止盈规则:
      • 盈利超10%:
          - 移动止盈到成本价
          - 确保至少不亏损
          
      • 盈利超20%:
          - 离场30%仓位（锁定利润）
          - 剩余70%继续持有
          
      • 盈利超30%:
          - 再离场30%仓位
          - 剩余40%继续持有
          
      • 盈利超50%:
          - 离场剩余40%仓位
          - 全部平仓
          
    最大回撤监控:
      • 实时计算当前回撤
      • 回撤 ≥ 15%: 触发警告，暂停加仓
      • 回撤 ≥ 20%: 强制止损，全部平仓
    ──────────────────────────────────────────────────────────────
```

## 四、最大回撤监控详细设计 ★ 核心新增

### 4.1 最大回撤定义

```yaml
最大回撤约束:

  马丁策略最大回撤: ≤ 20%

  回撤计算公式:
    当前回撤 = (当前权益 - 历史最高权益) / 历史最高权益 × 100%
    
    其中:
      • 当前权益 = 当前仓位市值 + 可用资金
      • 历史最高权益 = 历史上权益的最高值
      
  示例:
    • 历史最高权益: $100,000
    • 当前权益: $85,000
    • 当前回撤 = ($85,000 - $100,000) / $100,000 × 100% = -15%
```

### 4.2 最大回撤应对措施

```yaml
最大回撤应对措施:

  回撤 < 15%:
    • 状态: 正常
    • 应对措施: 正常执行
    • 可以: 入场、加仓、调整止盈止损
        
  回撤 15-20%:
    • 状态: 警告
    • 应对措施: 暂停加仓，仅执行止损
    • 可以: 止损、平仓
    • 不可以: 加仓、新开仓
        
  回撤 ≥ 20%:
    • 状态: 强制止损
    • 应对措施: 强制止损，全部平仓
    • 可以: 全部平仓
    • 不可以: 任何入场、加仓、调整
    • 后续: 等待下一次机会
```

### 4.3 最大回撤监控实现

```yaml
最大回撤监控实现:

  监控频率:
    • 正常状态: 每小时计算一次
    • 警告状态: 每15分钟计算一次
    • 强制止损状态: 实时监控
    
  监控实现:
    1. 记录历史最高权益
    2. 实时计算当前权益
    3. 计算当前回撤
    4. 根据回撤值触发应对措施
    
  记录到MEMORY:
    • 每次回撤 ≥ 15%，记录到MEMORY.md
    • 每次回撤 ≥ 20%，记录到MEMORY.md + 发送告警
    
  实时监控流程:
    ──────────────────────────────────────────────────────────────
    【步骤1: 计算当前权益】
      • 当前权益 = 当前仓位市值 + 可用资金
          
    【步骤2: 更新历史最高权益】
      • 如果当前权益 > 历史最高权益:
          - 更新历史最高权益 = 当前权益
          
    【步骤3: 计算当前回撤】
      • 当前回撤 = (当前权益 - 历史最高权益) / 历史最高权益 × 100%
          
    【步骤4: 触发应对措施】
      • 回撤 < 15%: 正常执行
      • 回撤 15-20%: 警告，暂停加仓
      • 回撤 ≥ 20%: 强制止损，全部平仓
    ──────────────────────────────────────────────────────────────
```

## 五、21事件库详细设计

### 5.1 事件类型

```yaml
21事件库:

  事件类型:
    • 黑天鹅事件 (如: 交易所暴雷、极端行情)
        - 严重程度: 高
        - 应对措施: 强制平仓，暂停交易
          
    • 监管政策突变 (如: 禁止加密货币交易)
        - 严重程度: 高
        - 应对措施: 强制平仓，暂停交易
          
    • 交易所异常 (如: 无法登录、无法交易)
        - 严重程度: 中
        - 应对措施: 暂停加仓，仅执行止损
          
    • 流动性危机 (如: 无法平仓、滑点巨大)
        - 严重程度: 高
        - 应对措施: 强制平仓，接受滑点
```

### 5.2 事件库维护

```yaml
21事件库维护:

  事件库初始化:
    • 预先定义21个风险事件
    • 每个事件包含: 事件类型、触发条件、严重程度、应对措施
    
  事件库更新:
    • 每次触发后记录到MEMORY.md
    • 每周回顾事件库
    • 根据A8验证结果调整事件库
    
  事件触发流程:
    ──────────────────────────────────────────────────────────────
    【步骤1: 检测事件触发】
      • A6监控检测21事件库
      • 如果触发 → 进入步骤2
          
    【步骤2: 评估事件严重程度】
      • 高: 强制平仓，暂停交易
      • 中: 暂停加仓，仅执行止损
      • 低: 注意观察，可以执行
          
    【步骤3: 执行应对措施】
      • 强制平仓 或 暂停加仓 或 正常执行
          
    【步骤4: 记录到MEMORY】
      • 记录事件名称、触发时间、应对措施
    ──────────────────────────────────────────────────────────────
```
