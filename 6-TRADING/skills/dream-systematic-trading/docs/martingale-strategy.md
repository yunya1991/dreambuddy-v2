## 一、马丁策略核心定位

### 1.1 策略定义

**马丁策略 (Martingale Strategy)**:
```
┌─────────────────────────────────────────────────────────────────────┐
│                    马丁策略核心思想                                    │
├─────────────────────────────────────────────────────────────────────┤
│                                                                   │
│  核心思想:                                                          │
│  "亏损时加仓，摊薄成本，等待反弹"                                  │
│                                                                   │
│  资金管理方式:                                                      │
│  分批建仓 + 阶梯加仓 + 严格止损                                    │
│                                                                   │
│  适用场景:                                                          │
│  • 趋势明确但波动大的市场 (合约马丁)                                │
│  • 低波动/震荡市场 (现货马丁)                                      │
│                                                                   │
│  风险:                                                              │
│  • 最大回撤约束: ≤ 20% ★ 核心新增                               │
│  • 资金管理能力要求高                                              │
│                                                                   │
└─────────────────────────────────────────────────────────────────────┘
```

### 1.2 策略类型

| 策略类型 | 适用场景 | 杠杆 | 风险 | 基础仓位 |
|:----------|:---------|:------|:-----|:----------|
| **合约马丁** | 高波动 + 趋势明确 | ≤ 5x | 高 | 20% |
| **现货马丁** | 低波动/震荡 | ≤ 2x | 低 | 25% |

## 二、马丁策略资金管理详细设计

### 2.1 四等份资金分配

```yaml
四等份资金分配:

  总仓位: 基础仓位 (合约马丁20% / 现货马丁25%)
  
  分配方式:
    Level 0: 入场仓位 = 50% 总仓位
    Level 1: 第一次加仓 = 30% 总仓位
    Level 2: 第二次加仓 = 20% 总仓位
    Level 3: 第三次加仓 = 0% (极限阶梯，不再加仓)
    
  示例 (合约马丁, 总仓位20%):
    Level 0: 入场 = 20% × 50% = 10%
    Level 1: 加仓 = 20% × 30% = 6%
    Level 2: 加仓 = 20% × 20% = 4%
    Level 3: 加仓 = 0% (不再加仓，等待反弹)
    
  累计仓位:
    Level 0: 10%
    Level 1: 10% + 6% = 16%
    Level 2: 16% + 4% = 20% (达到总仓位上限)
    Level 3: 20% (不再加仓)
    
  最大回撤约束: ≤ 20% ★ 核心新增
    • 实时计算当前回撤
    • 回撤 ≥ 15%: 触发警告，暂停加仓
    • 回撤 ≥ 20%: 强制止损，全部平仓
```

### 2.2 仓位计算公式 ★ 核心创新

```yaml
仓位计算公式:

  实际仓位 = 基础仓位 × (信号强度 / 100) × 风险系数
  
  其中:
    • 基础仓位: 根据策略类型确定
        - 合约马丁: 20%
        - 现货马丁: 25%
        
    • 信号强度: 0-100 评分
        - 强信号(≥70): 100%
        - 中信号(50-69): 60%
        - 弱信号(<50): 30% ★ 不下漏趋势
        - 无信号: 10% ★ 默认多头
        
    • 风险系数: 0.5-1.5
        - 根据波动率调整
        - 波动率 > 5%: 风险系数 = 0.5-0.8
        - 波动率 3-5%: 风险系数 = 0.8-1.2
        - 波动率 < 3%: 风险系数 = 1.2-1.5
        
    • 最大回撤约束: ≤ 20%
        - 实时计算当前回撤
        - 回撤 ≥ 15%: 触发警告，暂停加仓
        - 回撤 ≥ 20%: 强制止损，全部平仓

  示例计算:
    情景1: 强信号 + 合约马丁 + 波动率4%
      - 实际仓位 = 20% × (100/100) × 1.0 = 20%
        
    情景2: 中信号 + 现货马丁 + 波动率2%
      - 实际仓位 = 25% × (60/100) × 1.4 = 21%
        
    情景3: 弱信号 + 合约马丁 + 波动率6%
      - 实际仓位 = 20% × (30/100) × 0.6 = 3.6%
      - ★ 轻仓试探，不下漏趋势
        
    情景4: 无信号 + 现货马丁 + 波动率3%
      - 实际仓位 = 25% × (10/100) × 1.2 = 3%
      - ★ 最小仓位试探，保持市场存在感
```

### 2.3 资金使用限制

```yaml
资金使用限制:

  单层级仓位限制:
    • 单层级仓位 ≤ 账户20%
    • 累计马丁仓位 ≤ 账户60%
    • 预留至少40%作为安全垫
    
  最大回撤约束: ≤ 20% ★ 核心新增
    • 实时计算当前回撤
    • 回撤 = (当前权益 - 历史最高权益) / 历史最高权益 × 100%
    
  应对措施:
    • 回撤 < 15%: 正常执行
    • 回撤 15-20%: 触发警告，暂停加仓，仅执行止损
    • 回撤 ≥ 20%: 强制止损，全部平仓，等待下一次机会
```

## 三、马丁策略加仓阶梯详细设计

### 3.1 加仓阶梯计算

```yaml
加仓阶梯间隔计算:

  阶梯间隔 = max(波动率 × k, 最小间隔)
  
  其中:
    • 波动率: 20日波动率 (%)
    • k: 系数 (建议 0.3-0.8)
    • 最小间隔: 根据品种确定 (BTC: $500, ETH: $50)
    
  示例 (BTC/USDT):
    波动率 = 4.2%
    当前价 = $80,000
    k = 0.5
    阶梯间隔 = max(4.2% × 0.5 × $80,000, $500)
              = max($1,680, $500)
              = $1,680
```

### 3.3 动态调整规则 ★ v2.8 核心进化

```yaml
动态调整规则:

  核心原则: 基于价值评估做调整，追求更优价格
    
  ┌─────────────────────────────────────────────────────────┐
  │  评估类型      │  核心问题        │  决策依据              │
  ├─────────────────────────────────────────────────────────┤
  │  A4 价值验证  │  是否值得入场?    │  矛盾论验证 + 风险验证  │

  └─────────────────────────────────────────────────────────┘
    
  A4 价值评估 → 调整决策 ★★★★★:
    • A4 PASS → 确认阶梯有效，执行调整
    • A4 FAIL → 延迟调整，等待更好价格
    • A4 信号强度 → 影响调整幅度
      
  A5 价值评估 → 入场决策 ★★★☆☆:
    • A5 强信号 → 立即执行阶梯调整
    • A5 弱信号 → 等待A4确认后再调整
      
  A6 价值评估 → 时机监控 ★★★☆☆:
    • A6 发现更优价格 → 触发阶梯动态调整
    • A6 发现风险信号 → 触发A9风险评估
      
  A9 风险评估 → 离场决策 ★★★★★:
    • A9 强制离场: 基本面恶化 / 21事件触发
    • A9 条件离场: 移动止盈 / 时间止损
    • A9 风险预警 → 暂停阶梯调整，等待风险解除
    
  最大回撤监控: ★ 核心新增
    • 实时计算当前回撤
    • 回撤 ≥ 15%: 触发警告，暂停加仓和调整
    • 回撤 ≥ 20%: 强制止损，全部平仓，不再调整
```

## 四、马丁策略止盈止损详细设计

### 4.1 A9 L1 止损分层执行 ★ v2.8

```yaml
A9 L1 止损分层执行:

  加仓期 (Level 0-2):
    ──────────────────────────────────────────────────────────────
    止损策略:
      • 仅硬性保护止损（防止爆仓）
      • 不止盈（给市场波动空间）
      
    止损位计算:
      止损位 = 入场价 × (1 - 止损百分比)
      
      其中:
        • 弱信号: 止损百分比 = 2 × 20日波动率
        • 强信号: 止损百分比 = 1.5 × 20日波动率
        • 最大止损: 不超过20%（最大回撤约束）
        
    示例 (BTC/USDT, 多头):
      • 入场价: $79,800
      • 20日波动率: 4.2%
      • 弱信号止损百分比: 2 × 4.2% = 8.4%
      • 止损位: $79,800 × (1 - 8.4%) = $73,104
        
    最大回撤监控:
      • 实时计算当前回撤
      • 回撤 ≥ 15%: 触发警告，暂停加仓
      • 回撤 ≥ 20%: 强制止损，全部平仓
    ──────────────────────────────────────────────────────────────
    
  三层加满后 (Level 3):
    ──────────────────────────────────────────────────────────────
    止损策略:
      • 启用完整止盈止损
      • 移动止盈: 盈利超10% → 移动止盈到成本价
      • 分批离场: 盈利超20%/30%/50% → 分别离场30%/30%/40%
        
    移动止盈规则:
      • 盈利超10%:
          - 移动止盈到成本价
          - 确保至少不亏损
          
      • 盈利超20%:
          - 离场30%仓位（锁定利润）
          - 剩余70%继续持有
          
      • 盈利超30%:
          - 再离场30%仓位
          - 剩余40%继续持有
          
      • 盈利超50%:
          - 离场剩余40%仓位
          - 全部平仓
          
    最大回撤监控:
      • 实时计算当前回撤
      • 回撤 ≥ 15%: 触发警告，暂停加仓
      • 回撤 ≥ 20%: 强制止损，全部平仓
    ──────────────────────────────────────────────────────────────
```

### 4.2 止盈止损计算方法

```yaml
止盈止损计算方法:

  止损计算方法:
    止损位 = 入场价 × (1 - 止损百分比)

    止盈位 = 入场价 × (1 + 止盈百分比)

      • 强信号: 止盈百分比 = 5 × 20日波动率

  最大回撤监控: ★ 核心新增
    当前回撤 = (当前权益 - 历史最高权益) / 历史最高权益 × 100%
    
    应对措施:
      • 回撤 < 15%: 正常执行
      • 回撤 15-20%: 触发警告，暂停加仓，仅执行止损
      • 回撤 ≥ 20%: 强制止损，全部平仓
```

## 五、马丁策略最大回撤约束详细设计 ★ 核心新增

### 5.1 最大回撤定义

```yaml
最大回撤约束:

  马丁策略最大回撤: ≤ 20%
  
  回撤计算公式:
    当前回撤 = (当前权益 - 历史最高权益) / 历史最高权益 × 100%
    
    其中:
      • 当前权益 = 当前仓位市值 + 可用资金
      • 历史最高权益 = 历史上权益的最高值
      
  示例:
    • 历史最高权益: $100,000
    • 当前权益: $85,000
    • 当前回撤 = ($85,000 - $100,000) / $100,000 × 100% = -15%
```

### 5.2 最大回撤应对措施

```yaml
最大回撤应对措施:

  回撤 < 15%:
    • 状态: 正常
    • 应对措施: 正常执行
    • 可以: 入场、加仓、调整止盈止损
        
  回撤 15-20%:
    • 状态: 警告
    • 应对措施: 暂停加仓，仅执行止损
    • 可以: 止损、平仓
    • 不可以: 加仓、新开仓
        
  回撤 ≥ 20%:
    • 状态: 强制止损
    • 应对措施: 强制止损，全部平仓
    • 可以: 全部平仓
    • 不可以: 任何入场、加仓、调整
    • 后续: 等待下一次机会
```

### 5.3 最大回撤监控实现

```yaml
最大回撤监控实现:

  监控频率:
    • 正常状态: 每小时计算一次
    • 警告状态: 每15分钟计算一次
    • 强制止损状态: 实时监控
    
  监控实现:
    1. 记录历史最高权益
    2. 实时计算当前权益
    3. 计算当前回撤
    4. 根据回撤值触发应对措施
    
  记录到MEMORY:
    • 每次回撤 ≥ 15%，记录到MEMORY.md
    • 每次回撤 ≥ 20%，记录到MEMORY.md + 发送告警
```
