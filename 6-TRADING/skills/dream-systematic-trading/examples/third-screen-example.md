## 七、最大回撤监控示例 ★ 核心新增

### 7.1 最大回撤计算

```yaml
最大回撤监控:

  当前状态:
    当前权益: $102,700
    历史最高权益: $102,700 (更新)
    当前回撤: 0% (必须 ≤20%)

  回撤计算公式:
    当前回撤 = (当前权益 - 历史最高权益) / 历史最高权益 × 100%
                = ($102,700 - $102,700) / $102,700 × 100%
                = 0%

  应对措施:
    • 回撤 0% < 15%: 正常执行 ✅
    • 回撤 15-20%: 触发警告，暂停加仓
    • 回撤 ≥ 20%: 强制止损，全部平仓
```

### 7.2 最大回撤应对措施

```yaml
最大回撤应对措施:

  回撤 < 15%:
    • 状态: 正常
    • 应对措施: 正常执行 ✅
    • 可以: 入场、加仓、调整止盈止损

  回撤 15-20%:
    • 状态: 警告
    • 应对措施: 暂停加仓，仅执行止损
    • 可以: 止损、平仓
    • 不可以: 加仓、新开仓

  回撤 ≥ 20%:
    • 状态: 强制止损
    • 应对措施: 强制止损，全部平仓
    • 可以: 全部平仓
    • 不可以: 任何入场、加仓、调整
    • 后续: 等待下一次机会
```

## 八、示例总结

### 8.1 关键要点

1. **第三屏是执行层**: 使用 A4+A5+A6+A9，实时监控 + 执行交易
2. **A4-A9是核心工具**: A4战术验证 + A5决策执行 + A6情报监控 + A9离场决策
3. **弱信号不下漏趋势** ★: 轻仓试探（4.9%仓位），而非等待回落
4. **最大回撤监控** ★: 实时计算当前回撤，≥20%强制止损
5. **A9 L1分层执行**: 加仓期仅硬性保护止损，不止盈

### 8.2 风险提示

1. **价格偏离 >3%**: 中级告警，但未触发止损/止盈
2. **RSI超买**: 70，需警惕回调
3. **价量背离**: 触发，中风险
4. **最大回撤约束**: ≤ 20%，必须严格遵守
