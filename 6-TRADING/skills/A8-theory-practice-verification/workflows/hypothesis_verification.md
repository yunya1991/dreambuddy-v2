# A8 假说验证工作流程

> 版本：v1.0
> 用途：当A8发现矛盾后，提出假说并设计实验验证
> 触发条件：每日验证中发现矛盾 → 进入假说验证流程

## 一、工作流程概述

### 1.1 流程目标
- **假说提出**：基于发现的矛盾，提出合理的改进假说
- **实验设计**：设计可操作的验证实验
- **数据收集**：执行实验并收集数据
- **假说验证**：基于数据判断假说是否成立
- **理论完善**：验证通过的假说 → 完善A0/A7理论

### 1.2 流程时间安排（总时长：可变，通常2-5天）
```
第1天：假说提出与实验设计（120分钟）
第2-4天：实验执行与数据收集（自动化）
第5天：数据分析与假说验证（90分钟）
```

### 1.3 输入和输出
**输入**：
- A8每日验证报告中的矛盾分析
- 改进建议清单
- 历史实验数据（如果有）

**输出**：
- 假说验证报告（JSON+Markdown）
- 实验设计文档
- 验证通过/失败结论
- 理论完善提案（如果验证通过）

## 二、详细工作流程

### 第一阶段：假说提出（第1天，60分钟）

#### 步骤1.1：明确矛盾

#### 步骤1.2：提出假说
```
假说提出原则（基于毛泽东《实践论》）：
1. 假说必须解释已有证据
2. 假说必须可检验
3. 假说必须具体、明确
4. 假说应该尽可能简单（奥卡姆剃刀）

假说模板：
如果[改进措施]，那么[预期效果]，因为[理论依据]
```

### 第二阶段：实验设计（第1天，60分钟）

#### 步骤2.1：选择实验类型
```
实验类型选择树：
1. 回测实验：用历史数据验证
   - 适用：有充足历史数据
   - 优点：快速、低成本
   - 缺点：可能存在过拟合

2. 模拟盘实验：用模拟盘验证
   - 适用：需要实时市场数据
   - 优点：真实市场环境
   - 缺点：耗时较长

3. A/B测试：同时运行两个版本
   - 适用：可以并行运行
   - 优点：直接对比
   - 缺点：需要双倍资源

4. 影子模式：新版本影子运行，不实际执行
   - 适用：高风险改进
   - 优点：无风险
   - 缺点：无法验证实际效果
```

#### 步骤2.2：设计实验方案
```json
{
  "experiment_design": {
    "experiment_id": "E001",
    "hypothesis_id": "H001",
    "experiment_type": "回测实验",
    "duration": "10天（回测过去100天数据）",
    "sample_size": "100个交易信号",
    "control_group": "当前A2版本（无中间步骤记录）",
    "treatment_group": "改进A2版本（有中间步骤记录）",
    "success_criteria": "预测准确率提升10%以上",
    "risk_mitigation": "先在模拟盘验证，再应用到实盘"
  }
}
```

#### 步骤2.3：创建实验脚本
```python
# experiment_E001.py
"""
假说H001验证实验
对比：当前A2版本 vs 改进A2版本（增加中间步骤记录）
指标：预测准确率
"""
import pandas as pd
from datetime import datetime, timedelta

def run_experiment():
    # 1. 准备历史数据（过去100天）
    historical_data = prepare_historical_data(days=100)
    
    # 2. 运行控制组（当前A2版本）
    control_results = run_a2_current_version(historical_data)
    
    # 3. 运行处理组（改进A2版本）
    treatment_results = run_a2_improved_version(historical_data)
    
    # 4. 对比结果
    comparison = compare_results(control_results, treatment_results)
    
    # 5. 生成实验报告
    generate_experiment_report(comparison)

if __name__ == "__main__":
    run_experiment()
```

### 第四阶段：数据分析与假说验证（第5天，90分钟）

#### 步骤4.1：统计分析
```
统计检验选择：
1.  paired t-test：配对样本，前后对比
2.  independent t-test：独立样本，A/B测试
3.  chi-square test：分类数据，比例对比
4.  correlation analysis：相关性分析
```

#### 步骤4.2：假说验证判断
```
验证标准：
1. 统计显著性：p-value < 0.05
2. 效应大小：improvement > 最小可检测效应
3. 实际意义：改进是否有实际价值？
4. 可复现性：结果是否稳定？

验证结果：
- 验证通过：假说成立，可以完善到A0/A7理论
- 验证失败：假说不成立，需要提出新假说
-  inconclusive：结果不确定，需要更多数据
```

## 三、质量检查门禁

### 3.1 假说质量检查
```
□ 假说是否解释了已有证据？
□ 假说是否可检验？
□ 假说是否具体、明确？
□ 假说是否有理论依据？
```

### 3.2 实验设计质量检查
```
□ 实验类型选择是否合适？
□ 样本量是否足够？
□ 对照组和实验组是否可比？
□ 成功标准是否明确？
```

### 3.3 验证结论质量检查
```
□ 统计分析是否正确？
□ 验证标准是否合理？
□ 结论是否有数据支持？
□ 是否考虑了其他解释？
```
