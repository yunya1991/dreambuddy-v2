## 一、第一屏核心定位

### 1.1 战略层职责

**第一屏 = 周线决策 (战略层)**

```
┌─────────────────────────────────────────────────────────────────────┐
│                    第一屏：周线决策 (战略层)                          │
├─────────────────────────────────────────────────────────────────────┤
│                                                                   │
│  核心职责:                                                         │
│  1. 使用周线级别 A1-A3 进行战略分析                                 │
│  2. 判断多空方向（做多/做空/观望）                                   │
│  3. 选择双轨策略（合约马丁/现货马丁）                                 │
│                                                                   │
│  执行原则:                                                         │
│  ✅ 只做方向判断和策略选择                                           │
│  ❌ 不预设价格                                                     │
│  ❌ 不下单                                                         │
│                                                                   │
└─────────────────────────────────────────────────────────────────────┘
```

### 1.2 分析工具

**核心工具**: 周线级别 A1-A3 流水线

| 工具 | 用途 | 输出 |
|:-----|:-----|:-----|
| **A1 矛盾调查** | 识别周线级别多空矛盾 | 周线矛盾清单 + 矛盾优先级排序 |
| **A2 第一性原理** | 分析周线趋势动力来源 | 周线阻力最小方向 + 趋势动力评级 |
| **A3 沙盘推演** | 多情景周线推演 | 周线多情景概率分布 + 最优策略路径 |

## 二、第一屏执行流程详细设计

### 2.1 输入数据

```yaml
第一屏输入数据:

  # 周线技术分析
  - Elder 三屏周线数据 (EMA趋势、MACD、周线结构)
  - 周线支撑阻力位
  - 周线成交量分析
    
  # 基本面分析
  - 宏观经济环境 (利率、通胀、美元走势)
  - 行业/赛道基本面 (存储赛道: NAND/SSD供需)
  - ETF 资金流数据 (周级别)
    
  # 链上 + 市场情绪
  - 链上数据 (BTC/ETH/SOL 周线持仓变化)
  - 期权 IV 周级别数据
  - Fear & Greed Index (周平均)
```

### 2.2 A1 矛盾调查 (周线级别)

```yaml
A1 矛盾调查 (周线):

  矛盾识别维度:
    C1: 供需矛盾
      - 周线供应区 vs 周线需求区
      - 示例: BTC 周线供应区 $85,000 vs 需求区 $75,000
      
    C2: 多空矛盾
      - 多头驱动因素 vs 空头驱动因素
      - 示例: 多头=BTC ETF资金流入; 空头=美联储加息预期
        
    C3: 宏观矛盾
      - 宏观经济 vs 行业基本面
      - 示例: 宏观=美元走强; 行业=NAND需求增长
        
    C4: 技术矛盾
      - 周线技术指标之间的背离
      - 示例: 价格创新高但MACD背离
        
    C5: 资金流矛盾
      - 大资金流向 vs 散户情绪
      - 示例: 大资金流出但Fear & Greed指数高位
        
    C6: 时间周期矛盾
      - 周线趋势 vs 日线波动
      - 示例: 周线多头但日线回调
        
    C7: 隐性矛盾
      - 未被市场注意的关键矛盾
      - 示例: 衍生品未平仓合约异常

  输出:
    - 周线矛盾清单 (按优先级排序)
    - 主要矛盾的转化条件
    - 与日线级别的一致性检验
```

### 2.3 A2 第一性原理 (周线级别)

```yaml
A2 第一性原理 (周线):

  核心原理1: 市场总是沿着阻力最小方向运行
    • 分析周线趋势动力来源
    • 识别周线级别的主要矛盾转化点
    • 输出: 周线阻力最小方向 (多/空/震荡)

  核心原理2: 趋势具有延续性
    • 周线趋势一旦形成，大概率延续
    • 识别趋势动力评级 (强/中/弱)
    • 输出: 趋势动力评级 + 趋势延续概率

  周线阻力最小方向判定:
    • 多头阻力最小:
      - 周线EMA多头排列
      - MACD周线金叉
      - 成交量配合
      - 基本面支持
        
    • 空头阻力最小:
      - 周线EMA空头排列
      - MACD周线死叉
      - 成交量萎缩
      - 基本面恶化
        
    • 震荡 (阻力平衡):
      - 周线EMA纠缠
      - MACD周线横盘
      - 成交量低迷
      - 基本面中性

  输出:
    - 周线阻力最小方向: [多/空/震荡]
    - 趋势动力评级: [强/中/弱]
    - 趋势延续概率: [%]
```

### 2.4 A3 沙盘推演 (周线级别)

```yaml
A3 沙盘推演 (周线):

  多情景推演:
    情景A: 多头延续
      - 概率: [?]%
      - 触发条件: 周线EMA多头排列 + MACD金叉 + 基本面支持
      - 预期涨幅: [?]%
      - 风险点: 周线阻力位
        
    情景B: 空头延续
      - 概率: [?]%
      - 触发条件: 周线EMA空头排列 + MACD死叉 + 基本面恶化
      - 预期跌幅: [?]%
      - 风险点: 周线支撑位
        
    情景C: 震荡整理
      - 概率: [?]%
      - 触发条件: 周线EMA纠缠 + MACD横盘
      - 预期区间: [$XX,XXX - $XX,XXX]
      - 风险点: 突破方向不明

  最优策略路径:
    • 情景A概率 > 60%: 选择多头策略
    • 情景B概率 > 60%: 选择空头策略
    • 情景C概率 > 60%: 观望等待

  输出:
    - 周线多情景概率分布
    - 最优策略路径
    - 关键风险点
```

### 2.5 多空不对称性判断

```yaml
多空不对称性判断:

  情景A: 多头信号强 (周线评分 ≥ 70)
    → 选择多头策略
    → 可选: 多头合约马丁 / 多头现货马丁
    → 建议: 如果波动率高(>5%)，选择合约马丁; 如果波动率低(<3%)，选择现货马丁
      
  情景B: 空头信号强 (周线评分 ≥ 70)
    → 选择空头策略
    → 可选: 空头合约马丁 / 空头现货马丁
    → 注意: 现货马丁不支持做空，只能选择合约马丁
      
  情景C: 多空信号都弱 (周线评分 < 70)
    → 默认多头（除非周线明确看空）
    → 轻仓试探（10-30%仓位）
    → 等待周线信号明确
```

### 2.6 双轨策略选择

```yaml
双轨策略选择:

  【合约马丁策略】
    适用场景:
      • 高波动市场 (周线波动率 > 5%)
      • 趋势明确但波动大 (周线 EMA 排列清晰)
      • 合约杠杆交易
      
    特点:
      • 使用合约（多空双向）
      • 带杠杆（可调整，建议 ≤ 5x）
      • 收益放大，但风险同步放大
      
    资金管理:
      • 基础仓位: 20%
      • 最大杠杆: 5x
      • 最大回撤约束: ≤ 20%
        
  【现货马丁策略】
    适用场景:
      • 低波动/震荡市场 (周线波动率 < 3%)
      • 长期持有为主
      • 稳健增值需求
      
    特点:
      • 使用现货（不支持做空）
      • 无杠杆或低杠杆（≤ 2x）
      • 风险可控，适合稳健策略
      
    资金管理:
      • 基础仓位: 25%
      • 最大杠杆: 2x (仅限部分平台)
      • 最大回撤约束: ≤ 20%

  策略选择矩阵:
    | 周线状态 | 推荐策略 | 理由 |
    |----------|----------|------|
    | 高波动 + 趋势明确 | 合约马丁 | 收益放大 |
    | 低波动 + 震荡市 | 现货马丁 | 稳健增值 |
    | 高波动 + 方向不明 | 观望 | 风险控制 |
    | 趋势初期 | 现货马丁 | 积累筹码 |
    | 趋势加速期 | 合约马丁 | 趋势跟随 |
```

## 五、第一屏风险控制

### 5.1 最大回撤约束 ★ 核心新增

```yaml
最大回撤约束:

  第一屏职责:
    • 明确最大回撤约束: ≤ 20%
    • 将最大回撤约束传递到第二屏和第三屏
    • 如果回撤 ≥ 20%，强制平仓

  回撤计算公式:
    当前回撤 = (当前权益 - 历史最高权益) / 历史最高权益 × 100%
    
  应对措施:
    • 回撤 15-20%: 触发警告，暂停加仓
    • 回撤 ≥ 20%: 强制止损，全部平仓
```

### 5.2 观望情景处理

```yaml
观望情景处理:

  触发条件:
    • 周线评分 < 70
    • 多空信号都弱
    • 市场方向不明

  处理方式:
    • 默认多头（除非周线明确看空）
    • 轻仓试探（10-30%仓位）
    • 保持市场存在感
    • 等待周线信号明确

  退出观望条件:
    • 周线评分 ≥ 70
    • 方向明确
    • A1-A3验证通过
```
