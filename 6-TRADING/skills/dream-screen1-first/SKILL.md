# 第一屏: 周线决策 SKILL (dream-screen1-first)

> **SKILL 定位**: 三屏交易体系的**战略层**
>
> **核心职责**: 只做方向判断和策略选择，不预设价格、不下单
>
> **执行周期**: 每周一 08:00 (或手动触发)
>
> **版本**: v1.0 | 最后更新: 2026-05-16

---

## 一、SKILL 架构定位

### 1.1 在三屏体系中的位置

```

│              三屏交易体系调用链                            │
```

### 1.2 核心原则

| 原则 | 说明 |
|------|------|
| ✅ 只做方向判断 | 输出做多/做空/观望 |
| ✅ 只做策略选择 | 合约马丁 / 现货马丁 |
| ❌ 不预设价格 | 价格留给第二屏 |
| ❌ 不下单 | 执行留给第三屏 |
| ★ 默认多头 | 无明确信号时默认多头（除非周线明确看空） |

## 二、Phase 1: 数据采集 ★ 新增模块

### 2.1 过去一周交易记录调取

```yaml
交易记录获取流程:

  数据源优先级:
    1. OKX API 实时数据 (最权威)
    2. Episodes 本地记录 (A4/A5决策历史)
    3. 回测引擎模拟数据 (补充参考)

  ──────────────────────────────────────────
  【Step 1】OKX API 交易记录
    端点: /api/v5/trade/orders-history-archive
    参数:
      instType: SWAP
      instId: BTC-USDT-SWAP
      begin: {7天前时间戳}
      limit: 100
    封装脚本: scripts/okx_unified_toolkit.py
    
    输出字段:
      - ordId: 订单ID
      - side: buy/sell
      - posSide: long/short
      - sz: 成交数量
      - avgPx: 成交均价
      - pnl: 盈亏
      - state: 订单状态
      - cTime/ uTime: 创建/更新时间

  【Step 2】Episodes 本地决策记录
    路径: data/episodes/
    文件格式: A4_*.json, A5_*.json
    时间范围: 最近7天
    
    提取内容:
      - 决策时间戳
      - 方向判断
      - 评分结果
      - 门禁结果 (PASS/FAIL)
      - 实际执行情况

  【Step 3】数据整合为"实践上下文包"
    格式:
      practice_context:
        date_range: "{7天前} ~ {今天}"
        total_trades: N
        win_rate: X%
        total_pnl: +$XXX
        max_drawdown: X%
        last_decision: "{最近一次A4/A5决策摘要}"
        
    用途:
      - 传入A1-A3分析作为实践校验输入
      - 对比理论判断 vs 实际结果
      - 识别系统性偏差（如：连续看多但实际下跌）
  ──────────────────────────────────────────
```

## 三、Phase 2: A1-A3 分析流水线 ★ 核心

### 3.1 A1 矛盾调查 (周线级别) — 必须调用

```yaml
A1 矛盾调查 (周线):

  ★ 调用方式: 读取 dream-contradiction-theory SKILL 并执行

  分析维度 (C1-C7):
    C1: 供需矛盾 — 周线供应区 vs 需求区
    C2: 多空矛盾 — 多头驱动 vs 空头驱动因素
    C3: 宏观矛盾 — 宏观经济 vs 行业基本面
    C4: 技术矛盾 — 周线技术指标背离
    C5: 资金流矛盾 — 大资金流向 vs 散户情绪
    C6: 时间周期矛盾 — 周线 vs 日线趋势差异
    C7: 隐性矛盾 — 衍生品/链上异常信号

  ★ 实践数据注入 (新增):
    将 Phase 1 获取的 practice_context 注入 A1:
      - 过去一周实际交易盈亏 → 验证C2多空矛盾的实际表现
      - Episodes中的决策记录 → 检查是否存在认知偏差
      - 连续同方向错误 → 触发矛盾升级警告

  输出:
    - 周线矛盾清单 (按优先级排序)
    - 主要矛盾及其转化条件
    - 与日线一致性检验
    - ★ 实践偏差标记 (如有)
```

### 3.2 A2 第一性原理 (周线级别) — 必须调用

```yaml
A2 第一性原理 (周线):

  ★ 调用方式: 读取 dream-first-principles SKILL 并执行

  两大核心原理:
    原理1: 市场总是沿着阻力最小方向运行
      → 识别周线趋势动力来源
      → 判定阻力最小方向 (多/空/震荡)

    原理2: 趋势具有延续性
      → 周线趋势一旦形成大概率延续
      → 评定趋势动力评级 (强/中/弱)

  ★ 回测数据注入 (新增):
    将 Phase 1 的 backtest_context 注入 A2:
      - 过去12周各方向收益数据 → 验证"阻力最小方向"假设
      - 波动率序列 → 判断当前是否处于趋势延续期
      - 回测置信度 → 影响趋势动力评级

  阻力最小方向判定矩阵:
    多头阻力最小: EMA多头排列 + MACD金叉 + 成交量配合 + 基本面支持
    空头阻力最小: EMA空头排列 + MACD死叉 + 成交量萎缩 + 基本面恶化
    震荡平衡:     EMA纠缠 + MACD横盘 + 成交量低迷 + 基本面中性

  输出:
    - 周线阻力最小方向: [多/空/震荡]
    - 趋势动力评级: [强/中/弱]
    - 趋势延续概率: [%]
    - ★ 回测验证标注
```

### 3.3 A3 沙盘推演 (周线级别) — 必须调用

```yaml
A3 沙盘推演 (周线):

  ★ 调用方式: 读取 dream-tactical-validator SKILL (A3推演模式) 并执行

  多情景推演:
    情景A: 多头延续
      概率: [?]%  (★ 由回测数据校准)
      触发条件: 周线EMA多头排列 + MACD金叉 + 基本面支持
      预期涨幅: [?]%  (★ 参考回测历史分布)
      风险点: 周线阻力位

    情景B: 空头延续
      概率: [?]%
      触发条件: 周线EMA空头排列 + MACD死叉 + 基本面恶化
      预期跌幅: [?]% 
      风险点: 周线支撑位

    情景C: 震荡整理
      概率: [?]%
      触发条件: 周线EMA纠缠 + MACD横盘
      预期区间: [$XX,XXX - $XX,XXX]
      风险点: 突破方向不明

  ★ 贝叶斯概率校准 (新增):
    先验概率 = A3初始评估概率
    似然函数 = 回测数据中类似历史情景的频率
    后验概率 = 先验 × 似然 (贝叶斯更新)
    
    示例:
      A3初评: 情景A(多头)概率=60%
      回测显示: 类似周线形态下多头实际胜率=75%
      贝叶斯校准后: P(多头|证据) ≈ 71%

  最优策略路径:
    • 情景A > 60%: 选择多头策略
    • 情景B > 60%: 选择空头策略
    • 情景C > 60%: 观望等待 (轻仓试探)

  输出:
    - 周线多情景概率分布 (★ 贝叶斯校准后)
    - 最优策略路径
    - 关键风险点
    - ★ 回测依据引用
```

## 四、Phase 3: 决策与输出

### 4.1 多空不对称性判断

```yaml
多空不对称性判断:

  情景A: 多头信号强 (周线评分 ≥ 70)
    → 选择多头策略
    → 可选: 合约马丁 / 现货马丁

  情景B: 空头信号强 (周线评分 ≥ 70)
    → 选择空头策略
    → 仅合约马丁 (现货不支持做空)

  情景C: 多空信号都弱 (周线评分 < 70)
    → 默认多头（除非周线明确看空）
    → 轻仓试探（10-30%仓位）
    → 等待周线信号明确
```

### 4.2 双轨策略选择

| 周线状态 | 推荐策略 | 理由 |
|----------|----------|------|
| 高波动 + 趋势明确 | 合约马丁 | 收益放大 |
| 低波动 + 震荡市 | 现货马丁 | 稳健增值 |
| 高波动 + 方向不明 | 观望 | 风险控制 |
| 趋势初期 | 现货马丁 | 积累筹码 |
| 趋势加速期 | 合约马丁 | 趋势跟随 |
