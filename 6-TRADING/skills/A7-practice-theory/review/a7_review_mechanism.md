# A7 实践论复盘机制

## 复盘目标
 based on 毛泽东《实践论》的"真理的标准是实践"思想，建立系统的复盘机制，确保：
1. 实践结果被正确评估
2. 认识得到持续修正
3. 实践论应用不断优化

## 复盘频率

| 复盘类型 | 频率 | 触发条件 | 参与者 |
|:---|:---:|:---|:---|
| **每日复盘** | 每天 | A4/A5执行后 | A4/A5执行者 |
| **每周复盘** | 每周 | 周日 | A1-A5团队 |
| **每月复盘** | 每月 | 月末 | A1-A6 + 用户 |
| **事件驱动复盘** | 即时 | 重大亏损/重大盈利/战略错误 | A1-A6 + 用户 |

## 复盘内容框架

### 1. 每日复盘 (A4/A5执行后)

#### 复盘流程
```

    ↓

    ↓

    ↓

    ↓
记录经验教训
    ↓
确定下一步行动
    ↓
保存复盘记录到 .workbuddy/practice_reviews/daily/
```

### 4. 事件驱动复盘 (重大事件后即时)

#### 触发条件
1. 单笔亏损 >5%
2. 单笔盈利 >10%
3. 连续3笔亏损
4. 战略方向性错误
5. 重大市场事件导致策略失效

#### 复盘流程
```
重大事件发生
    ↓
立即暂停相关策略
    ↓
召回复盘会议 (A1-A6 + 用户)
    ↓
分析事件原因
    ↓
评估认识正确性
    ↓
确定认识修正方案
    ↓
确定策略调整方案
    ↓
保存事件复盘记录到 .workbuddy/practice_reviews/events/
```

## 复盘输出应用

### 1. 认识修正 → A1-A3
- 认识修正必须反馈到A1调研、A2分析、A3战略
- A1-A3必须根据复盘结果更新认识
- 更新后的认识必须重新验证

### 2. 经验教训 → A4-A5
- 经验教训必须应用到A4验证和A5执行
- A4-A5必须根据复盘结果优化实践
- 优化后的实践必须重新验证

### 3. A7门禁检查优化 → A7 SKILL
- A7门禁检查失败原因分析
- A7 SKILL优化建议
- A7 SKILL更新和测试

### 4. 实践论应用优化 → 整个系统
- 实践论应用效果评估
- 实践论应用优化建议
- 整个系统优化和升级

## 复盘记录管理

### 存储位置
```
.workbuddy/practice_reviews/
├── daily/       # 每日复盘记录
├── weekly/      # 每周复盘记录
├── monthly/     # 每月复盘记录
└── events/      # 事件驱动复盘记录
```

### 记录命名规范
```
daily_YYYYMMDD_HHMM.json
weekly_YYYY_WW.json
monthly_YYYY_MM.json
event_YYYYMMDD_HHMM_{event_type}.json
```

### 记录保留期限
- 每日复盘记录：保留30天
- 每周复盘记录：保留90天
- 每月复盘记录：永久保留
- 事件驱动复盘记录：永久保留
