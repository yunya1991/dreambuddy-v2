# 业务管理系统完整设计

> **版本**: v2.0
> **更新日期**: 2026-05-14

---

## 🏛️ 系统定位

业务管理系统涵盖Dream-MultiSkill除核心交易系统外的所有支持部门，提供：
- **人力资源**: 技能招聘、入职管理、试用期追踪
- **运营管理**: 流程优化、任务调度、系统健康
- **成本控制**: 预算控制、ROI评估、收益归因
- **行政支持**: 秘书服务、文档协作、会议管理

## 🏢 部门职责

### 1. 人力资源部 (HR)

**核心SKILL**: `dream-hr-recruitment`, `dream-performance-review`

**职责**:
- 技能市场搜索
- 候选人评估
- 技能安装与入职
- 试用期追踪
- 绩效考核
- PIP改进计划

**触发场景**:
- "需要一个XX技能"
- "现有技能不足"
- "功能缺失需要新技能"

### 2. 运营管理部 (COO)

**核心SKILL**: `dream-operation-director`, `dream-task-creator`, `auto-repair`

**职责**:
- 流程优化
- 任务分配
- 跨部门沟通
- 系统健康检查
- 自动化编排

**触发场景**:
- "协调多个部门"
- "流程卡顿需要疏通"
- "任务分配不明确"

### 3. 成本控制部 (CFO)

**核心SKILL**: `dream-cost-control`, `dream-execution-cost-model`

**职责**:
- 交易成本分析
- 算力预算控制
- ROI评估
- 收益归因
- 积分消耗监控

**触发场景**:
- "成本分析"
- "预算控制"
- "ROI评估"

### 4. 行政支持部 (Admin)

**核心SKILL**: `boss-secretary`, `腾讯文档`, `腾讯会议`, `notion`

**职责**:
- 邮件路由
- 报告生成
- 文档协作
- 会议管理
- 日报/周报

**触发场景**:
- "发送日报"
- "预约会议"
- "生成报告"

## 🔧 核心SKILL详情

### 1. dream-hr-recruitment

**功能**:
- 技能搜索与评估
- 候选人筛选
- 入职流程管理
- 试用期追踪

**目录结构**:
```
~/.workbuddy/skills/dream-hr-recruitment/
├── SKILL.md                      # SKILL定义
├── scripts/                     # 执行脚本
├── references/                  # 参考资料
└── templates/                   # 模板
```

**招聘流程**:
```
1. 需求分析 → "需要一个XX技能"
        ↓
2. 技能搜索 → find-skills
        ↓
3. 候选评估 → 匹配度评分
        ↓
4. 入职安装 → 安全审核
        ↓
5. 试用期 → 绩效追踪
        ↓
6. 转正/淘汰 → PIP改进
```

### 2. dream-performance-review

**功能**:
- 多部门技能评分
- KPI追踪
- PIP改进计划
- D级技能蒸馏移交

**目录结构**:
```
~/.workbuddy/skills/dream-performance-review/
├── SKILL.md                      # SKILL定义
├── scripts/                     # 执行脚本
├── kpi_*.json                   # KPI数据
└── distillation_handoff_*.yaml # 蒸馏移交
```

**绩效考核矩阵**:

| 维度 | 权重 | 评分标准 |
|------|------|----------|
| 执行效率 | 30% | 任务完成率/响应时间 |
| 质量评分 | 30% | 产出质量/准确率 |
| 成本控制 | 20% | 预算执行/积分消耗 |
| 团队协作 | 20% | 跨部门配合/沟通 |

**绩效等级**:

| 等级 | 评分范围 | 处理 |
|------|----------|------|
| A | 90-100 | 表扬/奖励 |
| B | 75-89 | 正常 |
| C | 60-74 | 观察 |
| D | <60 | PIP改进/蒸馏移交 |

### 3. dream-operation-director

**功能**:
- 流程优化建议
- 任务分配
- 跨部门协调
- 问题升级

**触发场景**:
- "协调XX和YY部门"
- "流程优化建议"
- "任务分配问题"

### 4. dream-cost-control

**功能**:
- 积分消耗分析
- ROI评估
- 收益归因
- 预算告警

**目录结构**:
```
~/.workbuddy/skills/dream-cost-control/
├── SKILL.md
├── scripts/
│   ├── cost_analyzer.py
│   └── roi_calculator.py
└── reports/
```

### 5. boss-secretary

**功能**:
- 邮件路由 (P0→A1-A5 / P1/P2→调研部 / P3→归档)
- 报告生成与投递
- 产物对齐管理
- Boss风控 (LOW/MEDIUM自动 / HIGH/CRITICAL确认)

**触发词**:
- "秘书"
- "发送报告"
- "邮件路由"

## 📋 核心流程

### 招聘流程

```
用户需求
   │
   ▼
┌────────────────────────────────────┐
│  dream-hr-recruitment              │
│  1. 需求分析                       │
│  2. 技能搜索 (find-skills)         │
│  3. 候选评估                       │
│  4. 安全审核 (skills-security-check)│
└──────────────┬─────────────────────┘
               ▼
        ┌─────────────┐
        │  入职安装   │
        └──────┬──────┘
               ▼
        ┌─────────────┐
        │  试用期追踪  │
        │  dream-    │
        │  performance│
        │  -review   │
        └─────────────┘
```

### 绩效考核流程

```
定时触发 (每日/每周)
        ↓
┌────────────────────────────────────┐
│  dream-performance-review          │
│  1. 数据采集                        │
│  2. 多维度评分                      │
│  3. 等级判定                        │
│  4. PIP触发判断                     │
└──────────────┬─────────────────────┘
               ▼
        ┌─────────────┐      ┌─────────────┐
        │  A/B/C等级  │      │  D等级      │
        │  继续跟踪    │      │  PIP改进    │
        └─────────────┘      └──────┬──────┘
                                     ▼
                              ┌─────────────┐
                              │ 蒸馏移交     │
                              │ D级→蒸馏部   │
                              └─────────────┘
```

### 成本监控流程

```
任务执行
   │
   ▼
┌────────────────────────────────────┐
│  dream-cost-control                │
│  1. 积分计算                        │
│  2. 预算对比                        │
│  3. ROI评估                        │
└──────────────┬─────────────────────┘
               ▼
        ┌─────────────┐
        │  告警判断   │
        │  >80%触发   │
        └──────┬──────┘
               ▼
        ┌─────────────┐
        │  告警通知   │
        │  Boss/Email │
        └─────────────┘
```
